# 002：reshape、转置与拼接不是同一类操作

## 发生场景

在 `reshape_practice/starter.py` 首次实现中，学习者：

1. 用 `X.reshape(n_features, n_samples)` 代替转置；
2. 用 `np.ndarray([1] * n_samples)` 尝试创建常数1列；
3. 调用 `np.ones(n_samples, 1)` 时没有把形状写成元组；
4. 调用 `np.concatenate(first, second, axis=1)` 时没有把数组组成序列，并把样本拼接轴写成了特征轴。

## 核心区别

- `reshape`：保持扁平元素顺序，只重新解释形状；
- `.T`：交换二维数组的两个索引轴，满足新数组`[j,i]`对应原数组`[i,j]`；
- `np.ones((n,1))`：内层元组描述二维形状；
- `np.concatenate((a,b), axis=0)`：内层元组是待拼接数组序列，`axis=0`延长样本轴；
- `np.concatenate((a,b), axis=1)`：延长特征轴。

## 最小反例

```python
X = np.array([[1, 2], [3, 4], [5, 6]])

X.reshape(2, 3)
# [[1, 2, 3],
#  [4, 5, 6]]

X.T
# [[1, 3, 5],
#  [2, 4, 6]]
```

两者形状相同，但数值位置不同；只检查`.shape`会漏掉这个错误。
