# 算法9：0/1背包、二维DP与物品恢复

## 题目

有 `n` 件物品，编号为 `0` 到 `n-1`。第 `i` 件物品的重量为 `weights[i]`，价值为 `values[i]`，每件物品最多选择一次。在总重量不超过容量 `capacity` 的条件下，求最大总价值，并恢复一组确定的物品下标。

输入第一行为 `n capacity`，随后恰好 `n` 行，每行是 `weight value`。重量必须为正整数，价值和容量必须为非负整数。

## 为什么需要二维状态

定义：

```text
dp[i][c] = 只看前i件物品、容量为c时能得到的最大价值
```

第 `i-1` 件物品只有两种选择：

```text
不选：dp[i-1][c]
选：  dp[i-1][c-weight] + value
```

“选”也从上一行 `i-1` 转移，所以同一件物品不会被重复使用。这正是 0/1 背包与无限硬币问题的关键区别。

## 平局与恢复规则

只有“选当前物品”能得到严格更大价值时才选择它；价值相同时保留“不选当前物品”。因为物品按下标从小到大加入DP，这会优先保留较早下标，并使恢复结果确定。

从 `dp[n][capacity]` 倒推：若 `dp[i][c] != dp[i-1][c]`，说明选择了下标 `i-1`，随后令 `c -= weights[i-1]`。

## 与一维写法的迁移

- 0/1背包压缩成一维时，容量必须从大到小遍历，避免本轮再次使用当前物品；
- 完全背包或无限硬币需要允许复用，容量通常从小到大遍历；
- 本题先保留二维表，因为它更容易手算、解释和恢复物品。

## 复杂度

时间复杂度 `O(n * capacity)`，二维表空间复杂度也是 `O(n * capacity)`。

## 学生任务

1. 先手算样例的前两行DP；
2. 完成 `build_knapsack_table`，先只返回最大价值；
3. 再根据相邻两行恢复下标；
4. 验证一件物品在容量足够时仍只能选择一次；
5. 测试容量0、没有物品能放入和价值平局；
6. 最后解释一维优化时为什么必须倒序遍历容量。
