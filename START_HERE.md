# 从这里开始学习

这不是参考答案导航，而是第一次学习会话的入口。当前目标只有一个：重新建立二维数组、形状和 `axis` 的直觉。

## 第一次学习会话（约30分钟）

### 1. 不运行代码，先预测（5～10分钟）

打开：

```text
02_numpy_basics/01_arrays_shapes_axes/README.md
```

根据下面的矩阵回答四个问题：

```python
X = np.array([
    [1.0, 10.0, 100.0],
    [2.0, 20.0, 200.0],
    [3.0, 30.0, 300.0],
])
```

1. `X.shape` 是什么？
2. `X[:, 1]` 的值和形状是什么？
3. `X.mean(axis=0)` 有几个数，分别表示什么？
4. `X.mean(axis=1)` 有几个数，分别表示什么？

把答案直接发给 Codex。不要因为不确定就先看输出；错误预测正是教学需要的诊断信息。

### 2. 运行演示并逐项对照（5分钟）

收到反馈后再运行：

```bash
python 02_numpy_basics/01_arrays_shapes_axes/guided_demo.py
```

不要只看“答案对不对”，要说明错误来自行列理解、索引降维还是 `axis` 方向。

### 3. 小步完成学生版本（10～15分钟）

打开：

```text
02_numpy_basics/01_arrays_shapes_axes/starter.py
```

一次只完成一个函数。每完成一个函数，先用2～3行小数组手动调用，不直接复制 `reference/solution.py`。

### 4. 本次结束条件

只有完成以下内容才结束第一次会话：

- 能说出 `(n_samples, n_features)` 两个维度的含义；
- 能区分“每个特征一个结果”和“每个样本一个结果”；
- 至少独立完成一个函数；
- 记录一个仍然不确定的问题。

## 不要做的事

- 不要一开始浏览所有参考答案；
- 不要连续看完四个 NumPy 专题却不写代码；
- 不要把“看懂输出”当作“能独立写出”；
- 不要因为旧目录里有类似代码就直接复制。
