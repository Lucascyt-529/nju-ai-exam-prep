# 神经网络1：感知机

## 先看形状

本专题统一使用：

```text
X: (n, d)       n个样本、每个样本d个特征
y: (n,)         每个样本一个标签，标签只能是-1或+1
weights: (d,)   每个特征一个权重
bias: 标量       一个偏置
scores: (n,)    每个样本一个线性得分
```

线性得分是：

```text
scores = X @ weights + bias
```

这里 `(n,d) @ (d,) -> (n,)`。预测规则固定为得分大于等于0时输出 `+1`，否则输出 `-1`。

## 逐样本更新

从全零权重和零偏置开始，按输入顺序逐个查看样本。若预测错误，则：

```text
correction = learning_rate * y_i
weights = weights + correction * X_i
bias = bias + correction
```

例如 `X_i=[-2,-1]`、`y_i=-1`、学习率为 `0.5`。初始得分为0，平局规则预测 `+1`，所以需要更新：

```text
correction = -0.5
weights = [0,0] + (-0.5) * [-2,-1] = [1,0.5]
bias = 0 - 0.5 = -0.5
```

每轮按固定顺序扫描所有样本；一轮没有发生更新时停止。固定初始化、样本顺序和平局规则使结果可复现。

## 能做什么，不能做什么

感知机可以在有限步内找到线性可分数据的一条分界线，但不保证对线性不可分数据收敛。因此训练函数还必须有 `max_epochs` 上限。XOR是最小的线性不可分反例之一。

更新次数不保证每一轮单调下降，不能把“历史曲线必须单调”写成错误断言。正确的性质检查包括：

- 线性可分样本最终全部预测正确；
- 收敛时最后一轮更新数为0；
- XOR等不可分数据不会谎报收敛，而是在上限停止；
- 输入数组、标签数组不会被原地修改。

## 学习顺序

1. 先完成 `guided_demo.py` 的运行前预测；
2. 手算一个样本的 `score`、预测和更新；
3. 补全 `decision_function` 与 `predict_perceptron`；
4. 再补全双层循环和停止条件；
5. 自己构造一维阈值不在原点的数据，确认 `bias` 确实被学习；
6. 最后用XOR解释为什么单个感知机不够。
