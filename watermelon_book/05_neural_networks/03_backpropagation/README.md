# 神经网络3：BP反向传播与数值梯度检查

## 先沿计算图倒着走

前向传播是：

```text
X -> z1 -> a1 -> z2 -> probabilities -> loss
```

反向传播从损失开始，沿相反方向使用链式法则：

```text
loss -> z2 -> W2和a1 -> z1 -> W1
```

仍沿用上一专题的形状：

```text
X:  (n,d)     y:  (n,1)
W1: (d,h)     b1: (h,)
a1: (n,h)
W2: (h,1)     b2: (1,)
p:  (n,1)
```

## 输出层梯度

sigmoid输出配合平均二元交叉熵时，两个导数可以合并成：

```text
dz2 = (p - y) / n                    (n,1)
dW2 = a1.T @ dz2                     (h,1)
db2 = sum(dz2, axis=0)               (1,)
```

`/n` 只放在 `dz2` 一次，后续链式计算自然继承平均因子，不要在每一层重复除以 `n`。

## 隐层梯度

```text
da1 = dz2 @ W2.T                     (n,h)
dz1 = da1 * a1 * (1-a1)              (n,h)
dW1 = X.T @ dz1                      (d,h)
db1 = sum(dz1, axis=0)               (h,)
```

每个梯度和对应参数形状完全相同。本专题让梯度字典继续使用参数名 `W1、b1、W2、b2`，因此可以逐键检查。

## 为什么还要数值梯度

损失下降不能证明梯度公式正确；转置、求和轴或广播出错时，某些小数据上仍可能偶然下降。对任意一个标量参数 `theta`，中心有限差分为：

```text
(loss(theta + epsilon) - loss(theta - epsilon)) / (2 * epsilon)
```

它很慢，但独立于反向传播公式，适合在小网络中验真。解析梯度和数值梯度的最大相对误差应很小。
