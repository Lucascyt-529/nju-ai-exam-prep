# 神经网络4：完整BP训练、预测与XOR

## 从正确梯度到训练循环

上一专题已经用有限差分证明梯度公式正确。本专题把它们放入全批量梯度下降：

```text
初始化参数
记录初始损失
重复epochs次：
    前向传播
    反向传播
    parameter = parameter - learning_rate * gradient
    记录更新后的损失
```

`loss_history` 长度固定为 `epochs + 1`，第0项是任何更新发生前的损失。这样才能公平回答“训练究竟有没有改进”。

## 为什么参数更新返回新字典

`apply_gradients` 不原地修改旧参数，而是返回全新的数组。这样可以：

- 在更新前后直接比较参数；
- 避免调试时旧缓存突然失效；
- 更容易检查输入不变性；
- 明确表达一次梯度下降产生了一个新状态。

## 预测形状

网络保留单输出神经元这一维：

```text
probabilities: (n,1)
labels:        (n,1)
```

概率大于等于阈值时输出1。默认阈值为0.5，因此概率恰好0.5时确定地归为1。

## XOR说明了什么

单个感知机无法表示XOR；含隐藏层的网络可以组合多个非线性边界。本专题固定：

- 4个隐藏神经元；
- sigmoid激活；
- 固定随机种子；
- 全批量梯度下降；
- 明确的学习率和轮数。

在这个可复现配置下，网络能拟合4个XOR样本。这证明该网络具有相应表达能力，也证明这一具体训练配置成功；它不等于任意初始化、学习率和轮数都保证到达全局最优。

## 训练性质怎样验证

- 历史长度必须是 `epochs+1`；
- 最终损失应显著低于初始损失；
- 固定种子和超参数应逐项复现参数与历史；
- XOR最终标签必须全部正确；
- 原始 `X、y、parameters、gradients` 均不得被原地修改。

损失不被写成对所有学习率都严格单调的普遍断言。学习率过大时，梯度下降完全可能震荡或发散。

## 学生任务

1. 先实现一次非原地参数更新；
2. 用 `epochs=0` 检查初始损失记录；
3. 在线性数据上检查最终损失和预测；
4. 再运行XOR并画出或打印若干损失检查点；
5. 改变随机种子，观察收敛速度而不是背固定答案；
6. 把学习率调得过大，解释为什么“梯度正确”仍不等于“训练一定稳定”。
