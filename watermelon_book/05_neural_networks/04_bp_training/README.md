# 神经网络4：标准BP、累积BP、预测与XOR

## 从正确梯度到训练循环

上一专题已经用有限差分证明梯度公式正确。本专题把同一组梯度公式放入两种更新流程。

| 教材名称 | 每次更新使用的数据 | 每轮参数更新次数 |
| --- | --- | --- |
| 标准BP | 当前一个训练样本 | `n_samples` |
| 累积BP | 整个训练集的平均梯度 | `1` |

已有的 `train_network` 保留为累积BP入口；`train_network_accumulated_bp`
是含义更明确的同等入口。标准BP使用 `train_network_standard_bp`。

## 累积BP训练循环

```text
初始化参数
记录初始损失
重复epochs次：
    前向传播
    反向传播
    parameter = parameter - learning_rate * gradient
    记录更新后的损失
```

## 标准BP训练循环

```text
初始化参数
记录初始的全训练集损失
重复epochs次：
    生成本轮样本顺序
    对顺序中的每个样本：
        只对该样本前向传播与反向传播
        立即更新参数
    记录本轮结束后的全训练集损失
```

标准BP对样本访问顺序敏感。`shuffle=False` 时每轮保持输入顺序，便于手算；
`shuffle=True` 时只创建一个随机数生成器并跨轮持续使用，避免错误地在每轮
重设同一个种子、得到完全相同的排列。初始化种子 `seed` 与打乱种子
`random_state` 分离，便于单独研究两种随机性的影响。

两种算法的 `loss_history` 长度都固定为 `epochs + 1`，第0项是任何更新发生前的损失。标准BP不把每个样本更新都塞入历史，而在每轮结束后计算一次全训练集损失，因此两种曲线可以按轮对齐；但一轮内的参数更新次数并不相同，比较时必须说明这一点。

## 为什么参数更新返回新字典

`apply_gradients` 不原地修改旧参数，而是返回全新的数组。这样可以：

- 在更新前后直接比较参数；
- 避免调试时旧缓存突然失效；
- 更容易检查输入不变性；
- 明确表达一次梯度下降产生了一个新状态。

## 预测形状

网络保留单输出神经元这一维：

```text
probabilities: (n,1)
labels:        (n,1)
```

概率大于等于阈值时输出1。默认阈值为0.5，因此概率恰好0.5时确定地归为1。

## XOR说明了什么

单个感知机无法表示XOR；含隐藏层的网络可以组合多个非线性边界。本专题固定：

- 4个隐藏神经元；
- sigmoid激活；
- 固定随机种子；
- 累积BP（全批量梯度下降）；
- 明确的学习率和轮数。

在这个可复现配置下，网络能拟合4个XOR样本。这证明该网络具有相应表达能力，也证明这一具体训练配置成功；它不等于任意初始化、学习率和轮数都保证到达全局最优。

## 训练性质怎样验证

- 历史长度必须是 `epochs+1`；
- 标准BP每轮恰好访问每个样本一次，`X` 与 `y` 必须使用同一下标；
- 固定初始化和打乱种子时，标准BP的参数、顺序与历史必须可复现；
- 最终损失应显著低于初始损失；
- 固定种子和超参数应逐项复现参数与历史；
- XOR最终标签必须全部正确；
- 原始 `X、y、parameters、gradients` 均不得被原地修改。

损失不被写成对所有学习率都严格单调的普遍断言。学习率过大时，梯度下降完全可能震荡或发散。

## 学生任务

1. 先实现一次非原地参数更新；
2. 用 `epochs=0` 检查初始损失记录；
3. 在线性数据上检查最终损失和预测；
4. 再运行XOR并画出或打印若干损失检查点；
5. 改变随机种子，观察收敛速度而不是背固定答案；
6. 把学习率调得过大，解释为什么“梯度正确”仍不等于“训练一定稳定”。
7. 在相同初始参数下各运行一轮标准BP和累积BP，解释为什么最终参数通常不同；
8. 把样本顺序反转后只运行标准BP，观察结果变化，并说明累积BP为何不受行顺序影响；
9. 将小批量大小设为1和全部样本数，分别说明它们与本专题两种BP的关系。
