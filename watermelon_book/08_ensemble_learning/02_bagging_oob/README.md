# 集成学习2：Bootstrap、Bagging与袋外评估

Bagging为每个基学习器从 `n` 个训练样本中有放回抽取 `n` 次。某些原样本会重复出现，另一些完全未被抽中；后者是该学习器的袋外（OOB）样本。

## 三组索引不要混淆

```text
bootstrap indices: 长度n，可重复，用于训练
unique in-bag:     真正出现过的原样本下标
OOB indices:       0..n-1中未出现在in-bag的下标
```

OOB不是额外划分的一份固定验证集：每个学习器的OOB集合不同。

## 本专题的透明基学习器

每个Bootstrap袋训练一个单特征决策树桩。若某袋只含一个类别，则训练确定的常数分类器。最终使用硬投票，票数相等归为 `+1`。

## OOB预测

对训练样本 `i`，只汇总没有抽到 `i` 的学习器：

```text
oob_score_i = mean(h_t(x_i) for t where i is OOB for learner t)
```

若没有任何学习器覆盖该样本，分数记为NaN并在OOB准确率中排除。增加学习器通常会提升覆盖率，但有限轮数不保证每个样本都覆盖。

## 学生任务

1. 从一组Bootstrap下标写出重复次数和OOB下标；
2. 验证同一随机种子复现完全相同的采样矩阵；
3. 手算三个基学习器的硬投票；
4. 为某个训练样本筛出合法OOB学习器；
5. 解释为什么不能用抽到该样本的学习器评价它；
6. 比较少量与大量学习器的OOB覆盖率。

参考实现通过不等于学习者已经独立掌握。
