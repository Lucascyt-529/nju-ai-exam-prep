# 集成学习3：随机子空间森林

随机森林在Bagging的样本随机性之外，还为每个树节点随机选择候选特征子集，再只在该子集中寻找最佳划分。

本专题使用单节点树桩，因此“每节点特征子采样”恰好等价于“每个学习器采样一次候选特征”。完整多层树应在每个内部节点重新采样，不能把本实现直接误称为完整工业随机森林。

## 两种随机性

```text
Bootstrap:       改变每个学习器看到的样本
feature subset:  改变每个节点允许比较的特征
```

只做Bootstrap时，多个强相关特征可能让所有树都选择同一主导特征；随机候选子集迫使一部分树探索其他特征，从而可能降低基学习器相关性。

## max_features

- 必须是 `1..d` 的整数；
- 每个学习器无放回抽取，子集内部升序保存；
- `max_features=d` 时退化为普通Bootstrap树桩Bagging；
- 相同种子同时复现Bootstrap矩阵与特征子集矩阵。

## 多样性观察

参考实现提供基预测矩阵 `(B,n)` 和平均成对预测相关系数。常数预测没有方差，相关系数未定义，会从均值中排除；若所有对都无定义则返回NaN。

## 学生任务

1. 区分样本采样和特征采样；
2. 检查每个特征子集无重复且大小正确；
3. 证明树桩选择特征一定属于自己的候选集合；
4. 手算多个树桩的硬投票；
5. 比较 `max_features=1` 与 `d` 的特征使用分布；
6. 解释相关性下降为什么不是每份数据上的数学保证。

参考实现通过不等于学习者已经独立掌握。
