# 集成学习5：Stacking学习法

## 为什么不能直接使用训练集预测

Stacking把每个基学习器的输出当作新的特征，再训练一个次级学习器。若基学习器先看过样本`i`的真实标签，又在样本`i`上生成次级特征，次级学习器看到的就是过分乐观的训练内预测，形成数据泄漏。

本专题用K折过程构造次级训练集：

1. 留出第`k`折；
2. 只在其余`K-1`折上训练每个基学习器；
3. 对第`k`折预测并填回原样本位置；
4. 每个样本恰好得到一次折外预测；
5. 用完整训练集重新训练每个基学习器，供未来新样本预测；
6. 用折外预测矩阵训练次级学习器。

若有`n`个样本和`B`个基学习器，次级特征矩阵形状是`(n,B)`。本参考闭环使用不惩罚截距的NumPy岭回归作为回归组合器；基学习器通过`fit(X,y)`和`predict(X)`接口接入，不依赖sklearn。

分类任务也遵守同一折外原则，但通常应使用类别概率或决策分数作为次级特征，而不是只使用硬标签。多分类时，一个基学习器可能贡献多列概率，因此不能机械套用本专题“一学习器一列”的回归接口。

## 必须分清的三组预测

| 预测 | 模型在哪里训练 | 用途 |
| --- | --- | --- |
| 训练内预测 | 全部训练样本 | 只能用于诊断，不能生成次级训练集 |
| 折外预测 | 不包含当前样本的折 | 训练次级学习器 |
| 新样本预测 | 完整训练集重训的基模型 | 部署时生成次级特征 |

## 学生任务

1. 生成K份互不重叠且并集为全部样本的验证下标；
2. 对一个基学习器手工追踪某个样本在哪一折成为验证样本；
3. 填充形状为`(n,B)`的折外预测矩阵；
4. 解释为什么每一折都必须创建新的基模型；
5. 给次级岭回归添加不惩罚截距的正则矩阵；
6. 用两个只输出不同特征列的简单基模型验证组合权重；
7. 构造“记住训练样本”的基模型，对比训练内预测与折外预测；
8. 说明最终预测为什么要使用完整训练集重训的基模型；
9. 思考分类概率为`(n,K)`时如何扩展次级特征矩阵。

不要先看参考实现。先画清楚某一折的训练下标、验证下标和次级矩阵填充位置，再写代码。

先运行`python guided_demo.py`观察训练内预测与折外预测的差别；演示故意使用会“记住训练样本”的模型，让泄漏现象直接可见。
