# 集成学习4：结合策略与多样性

## 常见结合策略

- 回归平均：对 `(B,n)` 个体预测按学习器轴平均；
- 加权平均：权重非负且和为1；
- 硬投票：每个分类器给标签，票数最多者获胜；
- 软投票：先平均 `(B,n,K)` 类别概率，再选最大概率类别；
- 学习法：用训练得到的次级学习器组合输出，必须用折外预测避免泄漏；完整Stacking闭环见`../05_stacking/`。

平局规则必须确定。本实现硬投票选择排序后较早类别，软投票选择较早类别下标。

## 二分类成对多样性

对两个分类器相对真实标签的正确性计数：

```text
n11: 两者都对   n00: 两者都错
n10: A对B错     n01: A错B对
```

由此计算：

```text
disagreement = (n10+n01)/n
double_fault = n00/n
Q = (n11*n00-n10*n01)/(n11*n00+n10*n01)
```

Q的分母为0时指标未定义，返回NaN而不是伪造0。

## 回归误差—分歧恒等式

等权平均回归满足逐样本恒等式：

```text
ensemble_squared_error
= mean(individual_squared_error)
  - mean((individual_prediction-ensemble_prediction)^2)
```

多样性项越大可能降低集成误差，但前提是个体误差没有同时变得过大；“越不同越好”并不成立。

## 多样性增强的四类入口

| 扰动对象 | 代表操作 | 主要风险 |
| --- | --- | --- |
| 数据样本 | Bootstrap或重加权 | 稳定学习器可能变化很小，少数样本可能被反复放大 |
| 输入属性 | 随机子空间 | 特征太少或冗余不足时会明显伤害个体准确性 |
| 输出表示 | 随机翻转标签、输出调制、ECOC | 扰动过强会把真实监督信号破坏掉 |
| 算法参数 | 不同初始化、结构或超参数 | 差异不保证有用，也可能只产生一批同样差的模型 |

参考实现生成可复现的样本下标、特征子空间、标签副本和参数种子，并把它们作为“扰动计划”返回，便于逐学习器审计。它不声称仅生成计划就完成了集成训练。

随机森林同时使用样本扰动和输入属性扰动；负相关法则显式鼓励神经网络个体参数/输出产生差异。不同机制可以组合，但最终仍要同时检查个体准确性、成对多样性和集成性能。

## 学生任务

1. 手算加权平均；
2. 手算硬投票和软投票；
3. 从标签写出四格计数；
4. 计算Q、分歧度和双错率；
5. 找到Q未定义的例子；
6. 验证一组回归预测的误差—分歧恒等式；
7. 解释Stacking为什么需要折外预测。
8. 判断Bagging、随机森林、标签翻转和随机初始化分别属于哪类扰动；
9. 检查Bootstrap矩阵中哪些样本重复、哪些样本袋外；
10. 更改子空间大小，解释多样性和个体准确性的可能变化；
11. 逐步增加标签翻转比例，说明为什么“更不一样”最终会伤害集成；
12. 用相同主种子重建扰动计划，验证所有下标、标签和参数种子完全一致。

参考实现通过不等于学习者已经独立掌握。
