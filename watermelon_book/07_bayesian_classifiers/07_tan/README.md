# 贝叶斯分类器7：TAN树增强朴素贝叶斯

朴素贝叶斯假设“给定类别后，各特征相互独立”。TAN（Tree Augmented Naive Bayes）只放松一步：每个特征除了依赖类别，还最多依赖另一个特征，并让这些特征依赖组成一棵树。

## 四步结构学习

1. 对每对特征计算给定类别后的条件互信息 `I(X_i; X_j | Y)`；
2. 把特征看作完全图结点，以条件互信息为边权；
3. 求最大带权生成树，保留最强的条件依赖；
4. 选择一个根并把树边定向，类别结点仍指向所有特征。

条件互信息越大，表示即使已经知道类别，两个特征之间仍保留较强依赖。它是结构评分，不是因果方向证据。

## 概率分解

若根特征为 `X_r`，其他特征的树父结点为 `pa_i`：

```text
P(Y, X) = P(Y) * P(X_r | Y) * product_i P(X_i | Y, X_pa_i)
```

参考实现用Laplace平滑并在对数域累加，避免许多小概率相乘下溢。当前教学版要求预测样本的离散取值在训练集中出现过；缺失值和未知类别应在数据处理阶段显式编码，不能静默当作普通值。

## 为什么示例能显示TAN价值

示例中类别0对应两个特征相同，类别1对应两个特征不同。对任一单独特征，两类边际分布都相同，朴素贝叶斯无法区分；但特征间“相同/不同”的条件依赖能被TAN树保留。

## 学生任务

1. 手算二值示例中一对特征的条件互信息；
2. 根据三特征权重矩阵手算Prim最大生成树；
3. 解释最大生成树为什么不是最短路树；
4. 写出根特征和非根特征的条件概率表形状；
5. 在对数域计算一个样本对每个类别的分数；
6. 对比只使用边际概率的朴素贝叶斯与TAN预测；
7. 更换根结点，说明树的无向边不变而分解方向改变。

参考实现通过不等于学习者已经独立掌握。
