# 决策树3：连续值与混合特征

## 连续值候选阈值

连续特征不能把每个数值直接当成一个离散分支。先对不同取值排序，只在相邻值中点尝试二分：

```text
values = [1, 1, 3, 5]
thresholds = [2, 4]
```

每个阈值产生 `<= threshold` 和 `> threshold` 两个非空训练子集，再计算信息增益、增益率或加权基尼指数。分数相同时选择更小的特征下标；同一特征内再选择更小阈值。

教材第4.4.1节的连续二分主线使用信息增益选择阈值，本专题的默认值保持这一规则。作为与第4.2.2节衔接的扩展，也提供连续二分增益率：分母是左右分支比例的固有值。使用该扩展构树时，先保留信息增益不低于候选平均值的特征，再在其中选择增益率最大者，避免偏爱信息量很少的极不平衡划分。连续特征内部按增益率选择自己的候选阈值，节点同时记录该阈值的信息增益，便于审计两步规则。

## 连续特征可以重复使用

离散特征沿一条路径划分后移除，因为它的取值已经决定。连续特征划分后仍保留：第一次可能区分 `x <= 2.5`，后代还可以用 `x <= 4.5` 形成另一个区间。

## 混合树节点

- 离散节点保存取值到子节点的映射，未见取值回退到当前多数类；
- 连续节点保存阈值，任何有限数值都能按左右分支处理，包括训练范围以外的值；
- 每个节点继续保存多数类作为确定性回退标签。

本专题支持`information_gain`、`gain_ratio`和`gini`。缺失值权重仍在独立专题处理；当前混合树要求输入有限，不能把“连续增益率已补齐”误写成“连续缺失值联合树也已完成”。

## 学生任务和迁移

1. 手算 `[1,1,3,5]` 的候选阈值；
2. 对每个阈值写出左右标签并计算信息增益；
3. 用单个连续特征构造三个类别区间，观察同一特征重复使用；
4. 把一列标为离散、一列标为连续，解释两种节点的不同；
5. 测试小于训练最小值和大于训练最大值的样本；
6. 解释为什么重复值之间不能产生候选阈值。
7. 对同一阈值手算左右分支固有值和增益率；
8. 构造高增益率但低信息增益的特征，解释两步筛选为什么可能排除它。
