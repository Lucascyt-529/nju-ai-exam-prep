# 决策树1：离散特征树

## 从三个量开始

标签集合中第 `k` 类比例为 `p_k`。

```text
entropy = -sum(p_k * log2(p_k))
gini = 1 - sum(p_k²)
```

按特征 `a` 划分后的信息增益为：

```text
gain = parent_entropy - sum(child_size / parent_size * child_entropy)
```

增益率再除以特征自身取值分布的熵 `intrinsic_value`。基尼指数则是各子节点基尼值的加权和。信息增益和增益率越大越好，基尼指数越小越好。

## 增益率的选择规则

直接选择最高增益率容易偏爱取值很少的特征。本专题采用常见的两步规则：

1. 先保留信息增益不低于候选平均值的特征；
2. 再从中选择增益率最高者。

完全相同时选择下标更小的特征，保证运行结果稳定。

## 树节点长什么样

参考实现使用普通字典，核心字段如下：

```text
is_leaf: 是否叶节点
prediction: 当前节点的多数类，也是回退标签
feature_index: 当前划分特征
children: 特征值到子节点的映射
```

每个内部节点都保留当前样本的多数类。当预测样本出现训练时未见过的特征值，就返回这个多数类，而不是报错或随意猜测。

## 递归停止条件

- 当前样本全属于同一类；
- 可用特征已经用完；
- 当前子集中所有剩余特征都只有一个取值。

若某个分支值在当前子集为空，创建一个父节点多数类叶子。训练时特征全集的可能取值会保留下来，以便明确验证这个规则。

## 形状与限制

- `X.shape == (n, d)`，本专题每列是离散数值编码；
- `y.shape == (n,)`，标签可以不连续；
- 同一个离散特征沿一条路径最多使用一次；
- 连续值、缺失值和剪枝尚未包含，不能把本实现标成完整决策树章节。

## 学生任务和迁移

1. 手算二分类各占一半时的entropy和gini；
2. 手算一个完美划分特征的信息增益；
3. 在运行前预测 `guided_demo.py` 会选择哪个根特征；
4. 完成 `starter.py` 中的度量，再实现选特征，最后写递归；
5. 用XOR数据验证根节点增益为0时仍可能需要继续划分；
6. 构造一个未见特征值，解释为何回退到当前节点多数类。
