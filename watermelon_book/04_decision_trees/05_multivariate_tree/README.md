# 决策树5：多变量线性划分

## 从轴平行到斜线边界

普通连续决策树节点只检查一个特征，例如：

```text
x_0 <= 2.5
```

多变量节点同时使用多个特征：

```text
X @ w <= threshold
```

二维时这是一条直线，高维时是一个超平面。它可能用一个节点表达普通树需要多个轴平行节点才能近似的斜边界。

## 可解释的确定性搜索

寻找最佳斜线本身是优化问题。本专题不调用sklearn或黑盒优化器，而使用清晰的小规模搜索：

1. 从每个坐标轴方向和类别均值差方向开始；
2. 对固定 `w`，将样本投影为 `z = X @ w`；
3. 只枚举相邻不同投影值的中点；
4. 选择加权基尼指数最低的阈值；
5. 依次对每个权重分量尝试 `+step` 和 `-step`，有改进就接受；
6. 一轮无改进时把步长减半。

方向每次归一化，并固定第一个非零分量为正，因此相同输入得到相同结果。

## 递归树

每个节点保存：

- `weights:(d,)`；
- `threshold`；
- 当前节点多数类和类别概率；
- 左右子树。

通过 `max_depth` 和 `min_samples_leaf` 控制复杂度。当前实现不含剪枝、缺失值或连续属性增益率的联合版本，它是多变量树的代表性线性划分实验。

## 学生任务和迁移

1. 手算 `w=[1,1]` 时二维样本的投影；
2. 写出所有候选投影中点；
3. 比较最佳单轴划分与斜线划分的基尼指数；
4. 从 `w=[1,0]` 出发，计算加入第二个分量后的变化；
5. 将数据整体旋转，检查线性节点能否继续找到纯划分；
6. 修改 `max_depth`，比较节点数和训练预测。
