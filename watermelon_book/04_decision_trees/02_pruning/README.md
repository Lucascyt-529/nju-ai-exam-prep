# 决策树2：预剪枝与后剪枝

## 为什么需要验证集

完整生长的决策树很容易记住训练数据中的偶然模式。剪枝必须使用独立验证集判断简化后是否更好，不能拿训练准确率同时完成建树和剪枝决策。

本专题沿用 [新决策树参考实现](../../../02_machine_learning/06_decision_tree/reference/solution.py) 的树结构和预测规则，新增两种剪枝函数。

## 预剪枝

每次准备划分前：

1. 只用当前训练子集选择最佳特征；
2. 比较“当前多数类叶节点”和“只进行当前一层划分”的验证准确率；
3. 只有划分准确率严格更高时才保留划分；
4. 若没有验证样本到达当前节点，保守地停止并生成叶节点。

验证集只决定是否保留划分，不参与信息增益、增益率或基尼指数计算。

## 后剪枝

先用训练集生成完整树，再自底向上处理每个内部节点：

1. 先剪枝它的子树；
2. 比较当前子树和替换成多数类叶节点后的验证准确率；
3. 若叶节点准确率不低于子树，就剪掉子树。

准确率相同时选择较小的树，这是本专题明确采用的保守规则。

## 不原地修改

`post_prune_tree` 深复制输入树。这样可以同时比较剪枝前后的节点数、预测和结构，也避免一次实验破坏下一次实验所需的完整树。

## 学生任务和迁移

1. 手算一次划分前后验证准确率；
2. 构造准确率相同的例子，预测预剪枝结果；
3. 画出后剪枝为什么必须自底向上的调用顺序；
4. 比较完整树、预剪枝树和后剪枝树的节点数；
5. 修改验证标签，说明为什么最佳划分特征本身仍只能由训练集确定；
6. 解释没有验证样本到达某个节点时为何采取保守叶节点。
