# 支持向量机8：替代损失与稀疏性

## 先统一横轴：带符号间隔

```text
margin = y * score
y in {-1,+1}
```

`margin<0`表示误分类，`margin>0`表示符号正确；SVM还要求`margin>=1`才在间隔之外。所有损失都只接收`margins:(n,)`，避免`y:(n,)`和`score:(n,1)`广播成`(n,n)`。

## 四种损失

```text
0/1:         1[margin < 0]
hinge:       max(0, 1-margin)
exponential: exp(-margin)
logistic:    log2(1+exp(-margin))
```

这里把对率损失除以`ln(2)`，所以`margin=0`时等于1，并对教材采用的严格负间隔0/1错误构成上界。常见逻辑回归使用自然对数；两者只差正比例常数，不改变单独经验损失的极小点，但会改变与正则项组合时`C`的等效尺度。

0/1损失不连续、非凸，直接优化困难。三种替代损失都是连续凸函数；hinge在`margin=1`不可导，使用次梯度。参考实现约定该点取右侧次梯度0。

## 为什么hinge产生稀疏性

```text
d hinge / d margin = -1, margin<1
                       0, margin>=1
```

间隔外样本的hinge损失和梯度都严格为0，因此它们不再推动当前经验损失，最终模型只依赖支持向量。指数和对率损失的梯度会随正间隔变小，却对任意有限间隔都不严格等于0，所以一般不具有同样的精确支持向量稀疏性。

“梯度很小”与“梯度为0”必须区分；用容差统计活跃样本只是数值诊断，不会把平滑损失在数学上变成稀疏模型。

## 数值稳定

- 对率损失用`np.logaddexp(0, -margin)`，避免直接计算`log(1+exp(...))`溢出；
- 对率梯度按margin正负分支计算稳定sigmoid；
- 指数损失在极大负间隔下确实可能溢出，参考实现给出明确错误；
- 比较不同损失的目标值时必须记住尺度不同，不能用同一个数值大小直接宣布谁更好。
