# 模型评估3：交叉验证与折外预测

## 每一折发生什么

```text
当前折的训练索引
→ 只用这些样本拟合预处理和模型
→ 在当前折的验证索引上预测
→ 保存这一折分数和折外预测
```

K折结束后，每个样本恰好得到一次“模型没有见过它时”的折外预测。折外预测可以按原始样本顺序合并，再计算整体度量。

## 为什么使用回调

本专题的 `cross_validation_scores()` 不绑定某个具体模型，而是接收：

- `fit_model(X_train, y_train)`；
- `predict(model, X_validation)`；
- `metric(y_validation, prediction)`。

这样同一个交叉验证框架可以用于后续线性回归、逻辑回归、决策树等模型。预处理必须放进 `fit_model` 所代表的折内流程，不能在调用交叉验证之前先对全数据拟合。

## 运行前预测

`guided_demo.py` 使用“始终预测当前训练折标签均值”的基线。运行前手算每折训练均值、验证预测和MSE。

## 候选方案选择

每个候选方案得到一组折分数。先报告均值和标准差，再根据度量方向选择：

- MSE、错误率：越小越好；
- accuracy、F1、AUC：越大越好。

不要看到一组分数均值略高就直接声称显著更好。统计比较检验将在后续专题单独建设。
