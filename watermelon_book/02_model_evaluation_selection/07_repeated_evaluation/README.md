# 模型评估7：重复留出与重复交叉验证

## 为什么一次划分不够

同一个模型只换一次训练集/测试集划分，分数就可能明显变化。原因不是算法代码突然变了，而是有限样本下，哪些样本被分到测试集本身带有偶然性。

周志华《机器学习》第2.2节指出：留出法通常要进行若干次随机划分，再报告多次评估结果的均值；同时也可以通过标准差观察划分带来的波动。K折交叉验证也可以重复，例如“10次10折”会进行100次训练和测试。

本专题把这两个过程写成可审计的完整函数：

```text
基础随机种子
→ 为每次重复生成独立、可复现的子种子
→ 生成本次训练/测试或K折索引
→ 只在训练部分拟合
→ 只在未见部分计分
→ 汇总全部运行的均值和波动
```

## 两种协议

### 重复留出

设重复次数为 `p`。每次重新随机划分一次训练集和测试集，训练一次模型，得到一个测试分数，因此一共拟合 `p` 次。

### 重复K折交叉验证

设重复次数为 `p`、折数为 `k`。每次重复都重新打乱并建立一套K折，每折训练一次，因此一共拟合 `p × k` 次。

`repeated_kfold_scores()` 返回形状为 `(p, k)` 的二维分数矩阵：

- 行：第几次重复；
- 列：该次重复中的第几折；
- `scores.mean(axis=1)`：每次重复的K折平均分；
- `scores.mean()`：全部训练运行的总体平均分。

这也是一次真实的NumPy形状练习。不要把 `(p, k)` 误当成 `(p * k,)` 后忘记“每次重复”这一层结构。

## 为什么返回索引和种子

参考实现不仅返回分数，还返回每次划分的：

- `seeds`：可以在另一台电脑上复现；
- `train_indices` / `test_indices`：可以验证训练和测试互斥；
- `folds`：可以验证每套K折恰好覆盖全部样本一次。

这让你能回答“模型到底见过哪些样本”，而不是只相信一个最终数字。

## 汇总结果怎么解释

`summarize_repeated_scores()` 同时给出：

- `overall_mean`：所有评估运行的平均分；
- `run_std`：所有单次运行分数的总体标准差；
- `repeat_means`：每次重复内部的平均分；
- `repeat_mean_std`：不同重复平均分之间的总体标准差；
- `fit_count`：实际模型拟合次数。

这里使用 `ddof=0` 的总体标准差，目的是描述当前这些重复结果的离散程度。它不能自动当成严格置信区间，因为不同划分会重复使用样本，评估结果并非彼此独立。

## 训练集比例的取舍

- 训练集更大：训练出的模型更接近使用完整数据时的模型；
- 测试集更大：测试误差通常更稳定，但留给训练的样本更少；
- 常见做法是把约三分之二到五分之四的样本用于训练，但具体比例仍要结合样本量和题目要求。

不能反复尝试随机种子，然后只汇报最好的一次。基础种子应在评估前固定，并完整保留所有重复结果。

## 无OJ自检清单

1. 同一个 `base_seed` 是否得到完全相同的划分和分数？
2. 改变 `base_seed` 后，划分是否真的变化？
3. 每次留出的训练/测试索引是否互斥且完整覆盖样本？
4. 每套K折中，每个样本是否恰好进入一次验证折？
5. 拟合函数收到的是否只有当前训练样本？
6. 预测形状是否严格为当前测试样本数对应的一维数组？
7. “10次10折”的拟合调用次数是否确实为100？
8. 输入数组在评估后是否保持不变？
