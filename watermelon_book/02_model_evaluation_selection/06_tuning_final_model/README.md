# 模型评估6：调参与最终模型

## 三种“参数”不要混在一起

- 模型参数：由训练算法从数据中学习，例如线性回归的权重；
- 超参数：训练前人为指定，例如多项式次数、学习率、正则强度；
- 搜索范围与步长：为了让调参可计算而人为离散化的候选集合。

实数超参数不可能逐个穷举。若三个超参数各取五个候选，笛卡尔积就有：

```text
5 * 5 * 5 = 125 个候选配置
```

从有限网格中选出的只是“当前候选里最好”，不能说成连续空间的理论最优值。

## 正确的数据流

```text
训练集 --逐个候选拟合--> 候选模型
验证集 --逐个候选打分--> 选定算法与超参数
训练集+验证集 --重新拟合一次--> 最终模型
测试集 --只评估最终模型一次--> 泛化性能估计
```

验证阶段训练出的最佳候选模型不是最终交付模型，因为它只使用了训练子集。超参数确定后，应合并训练集与验证集重新训练。测试集不能进入参数选择、预处理拟合或最终重训。

## 本专题的四个入口

- `cartesian_parameter_grid()`：生成有限参数笛卡尔积；
- `tune_on_validation()`：只用训练集拟合，只用验证集选择；
- `refit_selected_model()`：在训练+验证开发集上重训；
- `tune_refit_and_test()`：串起完整流程并只对最终模型计算一次测试分数。

所有候选必须包含相同参数名。相同验证分数时保留输入顺序靠前的候选，使结果可复现。度量方向必须明确：MSE越小越好，accuracy越大越好。

## 无OJ常见错误

1. 每试一个参数就看一次测试集，导致测试集变成验证集；
2. 选出最好参数后直接保存验证阶段模型，没有用全部开发数据重训；
3. 在划分前用全数据拟合缺失值、标准化或特征选择；
4. 把`higher_is_better`方向写反；
5. 只输出最好分数，不保存参数配置和全部候选记录；
6. 不计算网格大小，候选数指数增长后程序意外跑不完；
7. 把有限网格最优写成“全局最优”。

## 学习任务

1. 手算`{"degree":[1,2], "ridge":[0,0.1,1]}`共有多少配置；
2. 预测按参数名排序后前两个配置的字典内容；
3. 用常数回归器手算两个候选的验证MSE并选择；
4. 记录每次`fit_model`收到的样本编号，证明候选拟合不含验证/测试样本；
5. 证明最终那次拟合恰好收到训练集与验证集，仍不含测试集；
6. 把MSE改成accuracy，修正度量方向；
7. 构造验证分数平局，解释确定性规则；
8. 增加训练集标准化状态，确保候选阶段和最终重训阶段都只从当前拟合数据计算统计量。

参考实现通过只说明流程可运行，不表示学习者已经独立掌握。
