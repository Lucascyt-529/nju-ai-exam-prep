# 参考思路：发展脉络与应用判断

1. 推理规则需要作用于知识。若系统不知道领域事实、概念关系和常识，即使推理过程正确，也可能没有可用前提或得到无关结论。
2. 人工访谈、编码和维护大量规则成本很高，环境变化后规则还会过时；从示例数据中自动发现规律能缓解这一瓶颈，但会引入泛化与数据质量问题。
3. 一种方法的效果取决于数据规模、算力、优化、表示和任务。早期条件不足不等于思想无效；新条件也不保证它在所有问题上占优。
4. 符号路线偏规则、逻辑和概念结构；连接主义偏网络表示与分布式权重；统计路线偏样本、不确定性和泛化。现实方法可以组合这些思想，并非只能三选一。
5. 样本可定义为“某学生在某课程开课后一段时间的记录”；特征只能使用预测时已知的选课、早期作业等；标签是课程最终是否不及格；任务是二分类。需考虑漏掉高风险学生和误报带来的不同代价、类别不平衡、概率是否校准。把期末成绩或期末后统计量放进特征就是泄漏。
6. 没有异常标签就无法直接监督训练正常/异常分类器。可先定义为异常检测或单类学习，并通过人工复核、合成扰动或后续收集的故障记录建立评价集。
7. 至少需要灵敏度/召回率、特异度或误报率、阳性预测值、类别基率、概率校准、外部人群验证、置信区间、错误代价和人工复核流程。
8. 例如用旧学期学习行为训练的挂科预警模型，课程考核方式改变后失效。应监控输入特征分布、缺失率、预测分布、标签延迟后的性能和分群公平性。
9. 排名靠前的结果更容易被点击，用户点击还受标题吸引、设备和历史偏好影响；点击是带展示机制偏差的反馈，不等于无偏相关性标签。
10. 答案不限，但必须明确预测时点，不能把未来信息当特征；评价指标应与错误代价对应；部署约束至少包含一项模型分数之外的问题。
