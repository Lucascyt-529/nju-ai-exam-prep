# 聚类 5：DBSCAN 密度聚类

DBSCAN 不先规定簇数，而是用局部密度把高密度区域连接起来，并把稀疏区域标为噪声。

## 先分清三个概念

对样本 `i`，定义 `eps` 邻域：

```text
N_eps(i) = {j | distance(i, j) <= eps}
```

本仓库约定邻域包含样本自身。因此：

- `len(N_eps(i)) >= min_samples`：`i` 是核心点；
- 不是核心点，但落在某个核心点的邻域内：`i` 是边界点；
- 既不是核心点，也没有被任何核心点扩展到：`i` 是噪声点。

注意：`min_samples` 统计包含自己。若题目另有规定，必须先调整定义，不能机械套代码。

## 确定性的簇扩展

参考实现按样本下标从小到大寻找尚未扩展的核心点。每找到一个核心点，就进行队列扩展：

1. 当前核心点开启一个新簇；
2. 它邻域中的未归类样本加入该簇；
3. 邻域中若还有未扩展核心点，把它们加入队列；
4. 队列清空后，再寻找下一个核心点。

这样，同一批输入总会得到相同标签。一个边界点若同时邻接两个簇，会归入先扩展到它的簇；这正是 DBSCAN 中边界点可能受遍历顺序影响的地方。

## 形状

若 `X.shape == (n_samples, n_features)`：

```text
pairwise_euclidean(X).shape       == (n_samples, n_samples)
epsilon_neighborhoods(X, eps).shape == (n_samples, n_samples)
labels.shape                      == (n_samples,)
core_mask.shape                   == (n_samples,)
```

邻域布尔矩阵第 `i` 行表示哪些样本属于 `i` 的邻域。

## 学生任务

1. 手算一维小样本的距离矩阵与邻域矩阵；
2. 解释为什么邻域矩阵的对角线全为 `True`；
3. 根据邻域大小找出核心点；
4. 用队列完成核心点扩展；
5. 从最终标签区分核心点、边界点和噪声点；
6. 分别改变 `eps` 与 `min_samples`，解释簇和噪声怎样变化；
7. 构造一个非球形数据集，比较 DBSCAN 与 K-means 的差异。
