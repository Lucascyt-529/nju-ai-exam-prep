# 线性模型4：多分类拆分策略

## 为什么需要拆分

许多基础分类器一次只区分两个类别。多分类学习可以把一个 `K` 类问题拆成多个二分类问题，再把结果合并。

本专题使用一个透明的“两个类中心的线性打分器”演示拆分过程：

```text
score = X @ w + b
w = positive_mean - negative_mean
b = -0.5 * (||positive_mean||^2 - ||negative_mean||^2)
```

它只用于看清一对其余、一对一和ECOC的控制流程，不替代前面已经实现的对数几率回归或LDA。掌握拆分后，可以把基学习器换成其他二分类器。

## 一对其余（OvR）

对每个类别训练一个“当前类别 vs 其余全部类别”的二分类器。

若 `X.shape == (n, d)`，类别数为 `K`：

```text
weights.shape == (K, d)
biases.shape == (K,)
scores.shape == (n, K)
```

每行选择分数最大的类别。`np.unique(y)` 生成升序类别表，因此分数完全相同时，`np.argmax` 会稳定选择类别表中更靠前的标签。

## 一对一（OvO）

每两个类别训练一个二分类器，共有：

```text
K * (K - 1) / 2
```

个分类器。每个分类器只看它对应的两个类别。预测时每对类别投一票，票数最多者获胜。

本专题的确定性规则是：

1. 二分类分数大于0时投给类别对中的后一个类别；
2. 分数等于0时投给前一个类别；
3. 总票数相同时，选择升序类别表中更靠前的类别。

## 多对多与ECOC

多对多（MvM）每次把一部分类别作为正类、另一部分类别作为反类。最常见的
实现是纠错输出码（ECOC）。若有 `K` 个类别和 `M` 个二分类器，编码矩阵形状为：

```text
coding_matrix.shape == (K, M)
```

每一行是一个类别的目标编码，每一列定义一个二分类训练任务：

- `+1`：该类别样本作为正例；
- `-1`：该类别样本作为反例；
- `0`：三元码中的停用类，该列训练时不使用这个类别的样本。

每列都必须同时含 `+1` 和 `-1`，每个类别的编码行必须唯一。预测时，`M` 个
二分类器输出组成一个只含 `-1/+1` 的预测码，再与所有类别编码行比较，选择
海明距离或欧氏距离最小的类别。距离平局时沿用仓库确定性规则：选择升序类别表
中更靠前的原标签。

`make_ovr_coding_matrix` 和 `make_ovo_coding_matrix` 展示了两种已有策略都是
ECOC的特例：OvR是二元码；OvO是每列只启用两个类别、其余类别为0的三元码。

## 纠错能力与边界

对二元码，如果类别编码间的最小海明距离为 `d_min`，在理想的独立位错误假设下，
可保证纠正的错误位数为：

```text
floor((d_min - 1) / 2)
```

更长的码通常能增加码距，但也会训练更多分类器，并不保证实际效果一定更好：
不同二分类任务难度不同，各位错误也未必独立。三元码包含停用位，本专题仍可直接
计算和比较编码距离，但不把上述二元码纠错位数公式生搬硬套到三元码。

## 类别编码

标签不要求从0连续编号。例如原标签可以是 `10、20、30`。模型内部的类别下标是 `0、1、2`，最终预测必须映射回原标签，不能错误地直接输出内部下标。

## 学生任务和迁移

1. 对三个类别手写OvR的三个0/1标签向量和对应 `-1/+1` 编码矩阵；
2. 写出三个类别的OvO类别对和分类器数量；
3. 完成 `starter.py`，逐步打印分数矩阵和投票矩阵的形状；
4. 把标签从 `0、1、2` 改为 `10、20、30`，确认预测标签仍正确；
5. 构造三类各得一票的平票，解释最终规则；
6. 思考类别数增大时，OvR和OvO在分类器数量、每个分类器的数据量上的差别。
7. 为三类构造一个含0的三元码，逐列写出被选择的样本和二分类目标；
8. 给一个类别编码翻转一位，手算它到所有类别码的海明和欧氏距离；
9. 比较短码与长码的最小码距、分类器数量和理论纠错位数；
10. 解释为什么“编码理论码距更大”不等于真实分类准确率一定更高。
