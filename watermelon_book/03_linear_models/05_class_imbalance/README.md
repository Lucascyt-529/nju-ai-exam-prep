# 线性模型5：类别不平衡

## 先看清问题

若100个样本中有90个负类、10个正类，始终预测负类也有90%准确率，但正类召回率是0。因此类别不平衡时不能只看accuracy，还应检查混淆矩阵、precision、recall、F1、ROC/PR和具体误判代价。

本专题把处理方法分成三层，不能混为一谈。

## 1. 改变训练数据

- 随机欠采样：每类保留与最少类别相同的样本数；速度更快，但可能丢信息。
- 随机过采样：把每类补到最多类别的样本数；保留原样本，但重复样本可能加剧过拟合。

函数只返回索引。必须用同一组索引同时处理特征和标签：

```python
indices = random_undersample_indices(y_train, random_state=42)
X_balanced = X_train[indices]
y_balanced = y_train[indices]
```

采样只能在训练集内进行，不能先对完整数据采样再划分训练/验证集。

## 2. 改变训练损失权重

平衡类别权重定义为：

```text
weight_k = n_samples / (n_classes * count_k)
```

这样每个类别的样本权重总和相同，全体样本权重平均值为1。少数类单个样本权重更高，但数据本身没有被复制。

## 3. 改变最终决策

若假正例代价为 `C_FP`，假负例代价为 `C_FN`，最小化条件风险的正类阈值为：

```text
threshold = C_FP / (C_FP + C_FN)
```

假负例越昂贵，阈值越低，模型更愿意预测正类。阈值移动只改变标签决策，不应该把原概率文件悄悄改成另一组“概率”。

## 先验漂移修正

若训练/采样环境的正类先验为 `source_prior`，实际目标环境为 `target_prior`，并且只发生先验变化，可以修正后验优势比：

```text
target_odds = source_posterior_odds * target_prior_odds / source_prior_odds
```

这比单纯改阈值有更强假设：类别条件分布应保持稳定。参考实现同时给出概率修正和与 `0.5` 决策等价的原概率阈值，便于互相验证。
