# kNN

kNN（k近邻）没有显式的参数训练过程。它保存训练样本，预测时寻找距离查询样本最近的 `k` 个邻居：分类使用投票，回归使用平均。

## 算法概览

kNN 是典型的懒惰学习方法：训练阶段几乎只是保存数据，主要计算发生在预测阶段。它没有先从训练集概括出一组固定参数，而是直接利用训练样本回答每一次查询。

它依赖一个局部相似性假设：在选定的特征表示和距离度量下，彼此接近的样本往往具有相似输出。`k` 控制使用多大的邻域，小邻域更贴近局部细节，大邻域则更平滑。

kNN 简单但高度依赖距离质量。特征尺度、无关特征和高维空间都会影响邻居关系；维数增大后，样本之间的距离可能变得难以区分，这也是所谓维数灾难的一种表现。因此它既是实用基线，也是理解距离度量、局部学习和降维动机的重要算法。

## 输入与输出

- `X_train`：已知训练样本；
- `y_train`：训练样本的类别或连续目标；
- `X_query`：需要预测的新样本；
- `k`：参与预测的邻居数量；
- 分类输出：每个查询样本的类别；
- 回归输出：每个查询样本的数值预测。

当前先假定特征已经是可比较的数值，并且输入合法。实际任务中距离对尺度非常敏感，必须只用训练集拟合标准化参数，再复用到验证集和测试集。

## 欧氏距离

查询样本 `q` 与训练样本 `x` 的距离为：

```text
distance(q, x) = sqrt(sum((q_j - x_j) ** 2))
```

若有 `m` 个查询样本和 `n` 个训练样本，`pairwise_euclidean` 返回一个 `m × n` 距离表。第 `i` 行表示第 `i` 个查询样本到全部训练样本的距离。

## 完整算法流程

```text
训练数据 + 查询数据
-> 计算两两距离
-> 每行稳定排序
-> 取最近的 k 个训练下标
-> 分类投票 / 回归平均
-> 返回预测
```

“稳定排序”意味着距离相同时保留训练样本原顺序，这让结果可复现，也给投票平局提供确定规则。

## 函数职责

| 函数 | 输入 | 输出 | 下游用途 |
| --- | --- | --- | --- |
| `pairwise_euclidean` | 查询样本、训练样本 | 距离矩阵 | 找邻居 |
| `kneighbors` | 查询样本、训练样本、`k` | 邻居距离和下标 | 分类、回归 |
| `predict_classification` | 查询、训练、类别、`k` | 类别预测 | 分类评价 |
| `predict_regression` | 查询、训练、目标、`k` | 数值预测 | 回归评价 |

推荐实现：

```text
pairwise_euclidean
-> kneighbors
-> predict_classification
-> predict_regression
```

先让均匀投票和均匀平均正确，再处理距离加权。距离权重通常与距离成反比；若存在距离为0的训练样本，应直接使用这些重合样本，避免除0。

## 最小手算样例

```text
X_train = [[0], [1], [3], [4]]
y_train = [A, A, B, B]
X_query = [[0.5], [3.5]]
k = 2
```

对 `0.5`：

```text
distances = [0.5, 0.5, 2.5, 3.5]
nearest labels = [A, A]
prediction = A
```

对 `3.5`：

```text
distances = [3.5, 2.5, 0.5, 0.5]
nearest labels = [B, B]
prediction = B
```

若这是回归问题且 `y_train=[0, 2, 6, 8]`，两个查询点的均匀平均预测分别为1和7。

## k 如何影响模型

- `k` 很小：边界灵活，对噪声敏感，方差较高；
- `k` 很大：预测更平滑，可能忽略局部结构，偏差较高；
- `k` 不能通过测试集选择，应在验证集或交叉验证中比较；
- 二分类常用奇数 `k` 减少票数平局，但多分类仍可能平局。

分类使用 accuracy、F1 等指标；回归使用 MAE、MSE、RMSE 或 R2。只有训练数据没有“训练准确率下降过程”，因为 kNN 不迭代拟合参数。

## 常见错误

- 把特征轴当作样本轴求距离；
- 忘记开平方却把平方距离与真实距离混合使用；
- 选择了最远的 `k` 个样本；
- 邻居下标没有同步用于 `y_train`；
- 不同量纲特征未标准化，数值较大的特征支配距离；
- 用查询样本或测试样本拟合标准化参数；
- 分类平票依赖集合遍历顺序，结果不稳定；
- 距离加权时没有处理零距离。

## 自学闭环

1. 手算本页两个查询点的距离与预测；
2. 完成四个函数并运行 `check.py`；
3. 分别使用 `k=1、2、3`，预测一个位于类别边界附近的样本；
4. 构造票数平局，写清确定性规则；
5. 将某一特征乘100，观察未标准化预测为何改变；
6. 在验证集上选择 `k`，最后只报告一次测试结果。

能够解释“kNN 为什么没有普通意义上的训练循环”和“为什么标准化只能从训练集拟合”，才算掌握主线。
