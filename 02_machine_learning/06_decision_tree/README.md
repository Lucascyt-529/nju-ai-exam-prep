# 决策树

决策树通过一连串“如果某个特征取某个值”的判断把样本分到叶节点。训练的核心问题是：当前节点应该选择哪个特征，才能让分支后的类别更纯。

本入口先实现离散特征分类树。连续阈值、缺失值传播和剪枝放在主线完成之后。

## 算法概览

决策树采用分而治之的思想：从根节点开始选择一个属性测试，根据测试结果把数据划分到不同分支，再对每个分支重复同样过程。根到叶的一条路径可以看成一组条件的合取，整棵树则形成一套直观的分类规则。

建树过程需要回答两个问题：用哪个特征划分，以及何时停止。信息增益、增益率和基尼指数提供了不同的划分准则；纯节点、特征耗尽和样本不足等条件决定递归终止。

决策树能够表示非线性关系和特征交互，也便于解释，但完全生长的树容易过拟合。剪枝通过减少不可靠分支在拟合能力与泛化能力之间取得平衡，是理解模型复杂度控制的重要例子。

## 节点与叶子

一个内部节点至少需要保存：

- 当前选择的特征下标；
- 每个特征取值对应的子树；
- 当前节点的多数类，用于未见取值回退。

叶节点直接保存预测类别。递归建树时，每个子节点只看到到达该分支的样本。

## 衡量标签混乱程度

若类别比例为 `p_1, ..., p_K`：

```text
entropy = -sum(p_k * log2(p_k))
gini = 1 - sum(p_k ** 2)
```

纯节点只有一个类别，熵和基尼值都为0。类别越混合，它们越大。

用离散特征 `A` 划分后的条件熵是各分支熵的样本数加权平均：

```text
conditional_entropy = sum((n_v / n) * entropy(y_v))
information_gain = entropy(y) - conditional_entropy
```

信息增益越大越好。增益率用信息增益除以特征自身的取值熵，以减轻偏好取值很多的特征。基尼指数是划分后各分支基尼值的加权和，越小越好。

## 完整递归流程

```text
如果标签已经纯净：返回该类别叶子
如果没有可用特征：返回多数类叶子
否则评价所有候选特征
-> 选择最佳特征
-> 按该特征每个取值建立分支
-> 从可用集合删除该离散特征
-> 对每个分支子集递归建树
```

如果某个理论分支没有训练样本，应使用当前节点多数类作为叶子。预测时遇到训练中未见的取值，也回退到当前节点多数类。

## 函数职责

| 函数 | 作用 | 下游用途 |
| --- | --- | --- |
| `entropy`、`gini` | 衡量当前标签混乱度 | 划分评分 |
| `majority_label` | 得到稳定的多数类 | 停止与回退 |
| `information_gain` | 计算一个特征的信息增益 | 选特征 |
| `gain_ratio`、`gini_index` | 其他划分准则 | 比较树算法 |
| `choose_best_feature` | 遍历候选并稳定处理平局 | 递归建树 |
| `fit_discrete_tree` | 生成完整树 | 分类模型 |
| `predict_discrete_tree` | 逐样本沿分支下行 | 模型评价 |

推荐顺序：

```text
entropy -> gini -> majority_label
-> information_gain -> gain_ratio -> gini_index
-> choose_best_feature
-> fit_discrete_tree
-> predict_discrete_tree
```

## 最小手算样例

```text
feature = [0, 0, 1, 1]
y       = [0, 0, 1, 1]
```

父节点类别各占一半：

```text
entropy(y) = -(0.5*log2(0.5) + 0.5*log2(0.5)) = 1
```

按 `feature` 划分后，两个分支都只有一个类别，所以条件熵为0：

```text
information_gain = 1 - 0 = 1
```

这是完美划分，生成一层树即可准确预测训练样本。

## 停止、过拟合与剪枝

树一直生长可能记住训练噪声。除了纯节点和特征耗尽，还可以限制深度、叶节点最少样本，或使用验证集剪枝：

- 预剪枝：生长前判断划分是否改善验证表现；
- 后剪枝：先生成树，再自底向上比较子树与叶节点。

剪枝中的特征选择仍只能由训练数据决定，验证集只决定是否保留结构。扩展见 [决策树章节](../../watermelon_book/04_decision_trees/README.md)。

## 常见错误

- 熵中把样本数直接代替概率；
- 忘记按分支样本比例加权；
- 信息增益应最大化，却选了最小值；
- 基尼指数应最小化，却沿用信息增益方向；
- 多数类平局没有固定规则，结果不可复现；
- 子树递归时仍传入全部样本；
- 离散特征使用后没有从可用特征中删除；
- 信息增益为0就无条件停止，导致 XOR 这类需要继续组合划分的数据无法拟合；
- 预测遇到未见取值直接报错，没有节点回退标签。

## 自学闭环

1. 手算本页熵和信息增益；
2. 完成三个评分准则并解释优化方向；
3. 完成一层树，再扩展为递归树；
4. 构造两个同分特征，确认平局规则稳定；
5. 构造预测时未见取值，验证多数类回退；
6. 用 XOR 检查零增益根节点后是否仍能继续递归；
7. 最后比较完整树和剪枝树的训练、验证表现。

能手工追踪一个样本从根节点到叶节点，并解释每个停止条件，才算掌握主线。
