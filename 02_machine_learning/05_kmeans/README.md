# K-means

K-means 是无监督聚类算法。数据没有类别标签，算法根据样本之间的位置把它们分为 `k` 个簇，并用每个簇的均值作为中心。

## 算法概览

聚类试图在没有真实类别标签的情况下发现数据内部结构。K-means 属于原型聚类：每个簇用一个中心向量代表，样本归入距离自己最近的原型。

选择均值作为原型与平方欧氏距离密切相关。固定簇划分时，簇内样本的均值正是使平方距离和最小的中心；固定中心时，把样本分给最近中心又是最优分配。因此算法交替执行这两个步骤。

这种交替优化只能保证目标逐步不增，不能保证找到全局最优解，所以初始化会影响结果。K-means 还偏好近似球形、尺度相近的簇，对非凸形状、离群点和不同密度的数据可能表现较差。

## 优化目标

对每个样本 `x_i`，记所属中心为 `center[label_i]`。K-means 希望最小化簇内平方误差：

```text
SSE = sum(||x_i - center[label_i]|| ** 2)
```

这个值也常叫 inertia。它只能比较同一数据、同一预处理下的结果，而且随着 `k` 增大通常会下降，所以不能只凭训练 SSE 最小就选择最大 `k`。

## 两个交替步骤

K-means 反复执行：

```text
分配步骤：每个样本选择最近中心
更新步骤：每个中心变为所属样本的均值
```

每轮都不会增加当前目标，直到中心移动很小、标签不再变化，或达到最大迭代次数。

## 初始化为什么重要

目标函数不是凸函数，不同初始中心可能收敛到不同局部结果。K-means++ 的思想是：

1. 随机选择第一个中心；
2. 计算每个样本到最近已有中心的平方距离；
3. 距离越远，被选为下一个中心的概率越大；
4. 重复直到得到 `k` 个中心。

实际任务通常用多个随机种子重复运行，选择训练 SSE 最小的结果；随机种子要固定，才能复现实验。

## 函数职责

| 函数 | 输入 | 输出 | 下游用途 |
| --- | --- | --- | --- |
| `assign_labels` | 样本、当前中心 | 标签、到所属中心的平方距离 | 每轮分配与 SSE |
| `kmeans_plus_plus` | 样本、簇数、随机种子 | 初始中心 | 启动训练 |
| `fit_kmeans` | 数据和训练设置 | 中心、标签、SSE 历史、轮数 | 完整聚类模型 |

推荐顺序：

```text
assign_labels
-> kmeans_plus_plus
-> fit_kmeans
```

`fit_kmeans` 的循环中还要计算每个簇的新均值，并保存 `inertia_history`，用于观察目标是否下降。

## 最小手算样例

```text
X = [[0, 0], [1, 1], [9, 9], [10, 10]]
centers = [[0, 0], [10, 10]]
```

前两个样本离中心0更近，后两个离中心1更近：

```text
labels = [0, 0, 1, 1]
```

更新中心：

```text
center_0 = mean([[0,0], [1,1]]) = [0.5, 0.5]
center_1 = mean([[9,9], [10,10]]) = [9.5, 9.5]
```

再次分配时标签不变，算法即可停止。

## 空簇怎么理解

某轮可能没有样本分给某个中心，此时不能直接对空数组求均值。常见做法是把空簇中心重新放到当前误差最大的样本附近。重点不是死记某一种规则，而是保证：

- 中心数量仍为 `k`；
- 结果确定或由固定随机种子控制；
- 不产生 NaN；
- 后续仍可继续迭代。

## 如何选择 k

- 观察 SSE 的肘部，而不是机械选择最小值；
- 使用轮廓系数等同时考虑簇内紧密和簇间分离的指标；
- 结合题目语义，检查每个簇是否可解释；
- 比较多个随机初始化，避免偶然的较差局部结果。

特征尺度会直接影响距离。必须只用训练数据拟合标准化参数，并对后续数据复用。

## 常见错误

- 用普通距离而代码其他位置使用平方距离，SSE 定义不一致；
- 更新中心时没有按标签选择对应样本；
- 把旧中心原地改写后又用它计算本轮停止条件；
- 忘记处理空簇；
- 只运行一次随机初始化；
- 用真实类别标签直接命名簇，忽略簇编号本身没有类别语义；
- 仅凭训练 SSE 选择最大的 `k`；
- 标准化时使用全部数据统计量。

## 自学闭环

1. 手算本页一次分配和一次中心更新；
2. 完成 `assign_labels`，再完成初始化和训练循环；
3. 检查 `inertia_history` 是否非增；
4. 使用两个随机种子，比较最终中心和 SSE；
5. 将 `k` 从2改为3，解释 SSE 下降为什么不等于模型更合适；
6. 构造空簇并说明你的处理规则。

能够区分“簇标签只是编号”和“监督学习类别标签”，并能解释初始化敏感性，才算完成主线。
