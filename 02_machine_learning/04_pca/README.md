# PCA

PCA（主成分分析）是一种无监督降维方法。它寻找数据变化最大的相互正交方向，把原来的高维特征转换为较少的新特征，同时尽量保留信息。

## 算法概览

降维希望用更少的坐标描述原数据。PCA 采用线性变换，把原特征重新组合成一组彼此正交的主成分，并优先保留样本变化最大的方向。第一主成分保留最大方差，后续主成分在与已有方向正交的限制下继续保留剩余方差。

从另一角度看，在平方重构误差下，选择方差最大的低维子空间等价于让投影后再重构的数据尽量接近原数据。因此“最大方差”和“最小重构误差”是同一方法的两种理解。

PCA 不利用类别标签，所以保留下来的方向未必最有利于分类。它更适合数据压缩、去除线性冗余、可视化和作为下游模型的预处理；如果目标是利用标签寻找判别方向，后面的 LDA 更直接。

## PCA 在做什么

二维点云如果大致沿一条斜线分布，那么沿斜线方向变化很大，垂直方向变化很小。PCA 会把斜线方向选作第一主成分。只保留这一维，就能用较少坐标近似原数据。

PCA 不使用标签 `y`。它优化的是输入数据的方差保留或重构误差，不直接保证分类准确率提高。

## 训练与转换

给定训练数据 `X`：

```text
mean = X 在样本轴上的均值
centered = X - mean
covariance = centered.T @ centered / n
```

对协方差矩阵做对称特征分解：

```text
eigenvalues, eigenvectors = np.linalg.eigh(covariance)
```

将特征值从大到小排序，取前 `k` 个特征向量作为主成分。当前实现把每个主成分保存为 `components` 的一行，因此：

```text
Z = (X - mean) @ components.T
reconstructed = Z @ components + mean
```

`Z` 是低维表示，`reconstructed` 是从低维坐标恢复出的近似原数据。

## 完整流程

```text
训练数据
-> 计算训练均值
-> 中心化
-> 计算协方差矩阵
-> 特征分解并降序排列
-> 选择前 k 个主方向
-> 投影训练/验证/测试数据
-> 用重构误差或下游指标评价 k
```

验证集和测试集必须复用训练集的 `mean` 与 `components`，不能分别重新拟合 PCA，否则各数据集坐标系不同，也产生数据泄漏。

## 函数职责

| 函数 | 输入 | 输出 | 下游用途 |
| --- | --- | --- | --- |
| `fit_pca` | 训练数据、保留维数 | 均值、主成分、特征值等模型参数 | 后续转换 |
| `transform_pca` | 数据、训练均值、主成分 | 低维表示 `Z` | 可视化或下游模型 |
| `inverse_transform_pca` | `Z`、均值、主成分 | 重构数据 | 解释信息损失 |
| `reconstruction_mse` | 原数据、重构数据 | 一个误差值 | 比较保留维数 |

推荐顺序：

```text
fit_pca
-> transform_pca
-> inverse_transform_pca
-> reconstruction_mse
```

## 最小手算样例

```text
X = [[-2, -1],
     [-2,  1],
     [ 2, -1],
     [ 2,  1]]
```

两列均值都是0。第一列变化范围更大：

```text
第一列方差 = 4
第二列方差 = 1
协方差 = 0
```

因此第一主成分是水平方向。只保留一维时，第二列信息被丢弃，重构近似为：

```text
[[-2, 0], [-2, 0], [2, 0], [2, 0]]
```

每个样本第二个坐标误差平方为1，两个特征一起取平均，所以整体重构 MSE 为0.5。若保留两维，重构误差应接近0。

## 如何选择维数

- 看累计解释方差比例；
- 看验证集上的下游模型指标；
- 看重构误差是否在可接受范围；
- 可视化只需2或3维，但预测任务未必采用同样维数。

保留维数增加时，训练数据的重构误差不会上升，但这不等于下游验证性能一定提高。

## 常见错误

- 忘记中心化，第一主成分被数据整体位置影响；
- 对验证集和测试集分别计算均值或重新拟合主方向；
- 使用 `np.linalg.eig` 后忽略微小复数，而不是对称矩阵使用 `eigh`；
- 特征值没有按从大到小排序；
- 混淆 `components` 按行还是按列保存，矩阵乘法方向错误；
- 把 PCA 当成有监督方法，声称它一定保留最有分类能力的方向；
- 把降维后的 `Z` 与重构后的高维数据混为一谈。

## 自学闭环

1. 手算本页矩形点集的均值、协方差和第一主方向；
2. 完成四个函数并运行 `check.py`；
3. 分别保留一维和二维，比较重构误差；
4. 将全部样本整体平移，验证低维相对结构不变；
5. 在训练集拟合 PCA，将同一参数应用到验证集；
6. PCA 后接 kNN，使用验证集共同选择维数和 `k`。

能区分“拟合 PCA”“投影到低维”和“从低维重构”这三个动作，才算掌握主线。
