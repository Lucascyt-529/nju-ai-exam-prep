# 随机森林

随机森林在 Bagging 的样本随机性之外，再加入特征随机性。每棵树或每个节点只从随机候选特征中寻找最佳划分，使基础学习器之间更不相同。

本入口使用“Bootstrap + 随机特征子空间 + 决策树桩”的小型实现，先看清随机森林的核心机制，再扩展到完整树。

## 算法概览

随机森林以决策树为基础学习器。Bagging 已经通过 Bootstrap 改变每棵树看到的训练样本，但当某些特征特别强时，不同树仍可能反复选择相同划分，导致预测高度相关。

随机选择候选特征进一步打破这种相似性。单棵树可能因此略弱，但如果树之间的错误更不相关，集成后的多数投票反而可能更稳定。这体现了集成学习中准确性与多样性的权衡。

随机森林通常不需要像距离模型那样依赖特征缩放，能够处理非线性和特征交互，并可利用 OOB 数据评价。但模型解释比单棵浅树困难，特征重要性也只能表示关联贡献，不能自动解释因果关系。

## 为什么还要随机特征

如果某个特征特别强，普通 Bagging 中许多树可能反复选择它，基础预测高度相关。相关模型即使很多，投票带来的收益也有限。

随机限制候选特征会让部分学习器探索其他划分：

```text
较小 max_features
-> 基础学习器差异通常增大
-> 相关性通常降低
-> 单棵模型可能变弱
```

最终效果取决于“单个学习器能力”和“学习器多样性”的平衡，不能机械地认为 `max_features` 越小越好。

## 完整流程

```text
为每个学习器 Bootstrap 采样训练行
-> 为每个学习器采样候选特征子集
-> 只在候选特征中训练树桩
-> 收集全部基础预测
-> 多数投票得到最终类别
```

完整随机森林通常在树的每个节点重新采样候选特征；当前小实现每个树桩只有一个节点，因此“每个学习器一组候选特征”正好展示该机制。

## 函数职责

| 函数 | 输入 | 输出 | 下游用途 |
| --- | --- | --- | --- |
| `sample_feature_subsets` | 特征数、学习器数、候选数、种子 | 每个学习器的候选特征下标 | 随机子空间 |
| `fit_random_subspace_forest` | 数据和集成设置 | 树桩、Bootstrap 与特征记录 | 完整训练 |
| `base_predictions` | 模型、新数据 | 每个学习器的预测矩阵 | 投票与多样性分析 |

参考实现还提供平均决策分数、最终标签和基础预测相关性。主线先完成 starter 中三个函数。

## 最小样例

```text
n_features = 4
n_estimators = 3
max_features = 2
random_state = 0
```

固定种子得到：

```text
feature_subsets = [[2, 3],
                   [0, 1],
                   [0, 3]]
```

第一个树桩只能比较特征2和3，不能使用特征0或1；第二个只能比较0和1。即使训练行相同，它们也可能学到不同划分。

若3个基础学习器对某个样本预测为：

```text
[-1, 1, 1]
```

平均得分为 `1/3`，最终多数票为 `1`。

## 如何评价

- 使用验证集或 OOB 指标比较 `n_estimators` 和 `max_features`；
- 查看基础预测相关性，理解多样性是否真的增加；
- 学习器数量增加通常让结果更稳定，但训练和预测开销也增大；
- 随机森林不要求先标准化特征才能比较轴向阈值，但缺失值等数据处理仍需明确；
- 特征重要性只能谨慎解释，不能直接当作因果关系。

## 常见错误

- 特征子集使用有放回采样，导致同一候选集合内重复特征；
- `max_features` 超过总特征数；
- 采样了特征子集，但训练树桩仍遍历全部特征；
- 每个学习器使用完全相同的 Bootstrap 和特征集合；
- 基础预测矩阵把样本轴和学习器轴放反；
- 最终分类对树编号平均，而不是对 `-1/1` 预测投票；
- 只追求最低相关性，忽略单棵模型已接近随机猜测；
- 把当前随机子空间树桩直接等同于工业级完整随机森林。

## 自学闭环

1. 根据固定种子核对候选特征子集；
2. 确认每个树桩只使用自己的候选特征；
3. 打印基础预测矩阵并手算一个样本的最终票；
4. 比较 `max_features=1` 与使用全部特征时的基础预测相关性；
5. 同时比较单个学习器能力和最终验证结果；
6. 用流程图说明随机森林比 Bagging 多了哪一层随机性。

能解释“降低相关性为什么可能改善平均结果，但弱学习器也不能太弱”，才算掌握主线。
