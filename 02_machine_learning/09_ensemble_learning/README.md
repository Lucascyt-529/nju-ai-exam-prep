# 集成学习

集成学习把多个基础学习器组合起来。关键不只是“模型数量多”，而是基础学习器既要有一定正确性，又要犯不同的错误。完全相同的模型重复投票不会产生新的信息。

## 算法概览

集成学习的基本目标是通过组合多个学习器获得比单个学习器更好的泛化性能。要产生有效组合，基础学习器既不能弱到近似随机猜测，也不能彼此完全相同；准确性与多样性需要同时考虑。

常见路线有两类。Boosting 串行生成学习器，让后续模型更关注前面没有处理好的样本；Bagging 则通过不同训练样本并行生成学习器，再用平均降低波动。随机森林进一步引入特征随机性，增加树之间的差异。

组合策略本身也很重要：分类可以投票，回归可以平均，还可以学习一个次级模型进行组合。当前先掌握三种代表算法，再进入 Stacking 和多样性度量。

## 三条主线

| 顺序 | 算法 | 模型关系 | 随机性或关注点 | 评价 |

## 先建立的统一认识

```text
训练多个基础学习器
-> 收集每个学习器的预测
-> 分类投票 / 回归平均 / 加权组合
-> 用独立数据评价组合模型
```

- Boosting 主要通过串行纠错降低偏差；
- Bagging 主要通过平均多个不稳定模型降低方差；
- 随机森林在 Bagging 上降低树之间的相关性；
- 集成规模属于超参数，不应根据测试集选择；
- 基础模型训练准确率高，不代表组合后一定泛化更好。

## 推荐学习顺序

1. 先掌握单个决策树桩如何预测；
2. 学 AdaBoost 的样本权重与加权投票；
3. 学 Bootstrap、基础预测矩阵和普通投票；
4. 理解 OOB 中“哪些模型没看过当前样本”；
5. 再加入随机候选特征，比较基础模型相关性。

Stacking、多样性度量和更完整的推导见 [集成学习扩展区](../../watermelon_book/08_ensemble_learning/README.md)。三个子专题都完成后，应能不用代码解释它们的训练依赖关系和评估方式。
