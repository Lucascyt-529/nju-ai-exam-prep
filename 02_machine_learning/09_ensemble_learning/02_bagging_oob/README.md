# Bagging 与 OOB

Bagging 对训练集进行多次 Bootstrap 采样，在每份重采样数据上独立训练一个基础学习器，再通过投票或平均组合预测。它特别适合决策树这类对训练数据扰动敏感的模型。

## 算法概览

某些算法对训练数据的细小变化很敏感，例如决策树可能因为少数样本变化而选择完全不同的根节点。这类学习器方差较大。Bagging 通过对多个不同训练集上得到的模型取平均，使偶然波动相互抵消。

Bootstrap 用有放回采样模拟从原数据分布重新取样。每个基础学习器看到的样本集合不同，因此会形成差异；所有学习器又使用同一种训练方法，使组合仍围绕同一任务。

Bagging 主要降低方差，通常不会像 Boosting 那样让后一轮专门纠正前一轮。OOB 评价则利用每次采样自然留下的未见样本，是 Bootstrap 机制带来的附加能力。

## Bootstrap 采样

若训练集有 `n` 个样本，一份 Bootstrap 样本会从 `0...n-1` 中有放回地抽取 `n` 次：

```text
原下标：0, 1, 2, 3, 4
一次可能采样：[4, 3, 2, 1, 1]
```

下标1出现两次，下标0没有出现。没有被某个基础学习器采到的训练样本称为该学习器的袋外样本（OOB）。

## 完整流程

```text
生成 B 份 Bootstrap 下标
-> 每份数据独立训练一个树桩
-> 收集形状为 (B, n_query) 的基础预测
-> 分类取多数票 / 回归取平均
```

各基础学习器之间没有前后依赖，因此可以独立甚至并行训练。这与 AdaBoost 的串行重加权不同。

## OOB 评价

评价训练样本 `i` 时，只汇总那些 Bootstrap 训练数据中没有包含 `i` 的学习器：

```text
oob_score_i = mean(没有见过样本 i 的学习器预测)
```

如果某个样本尚未被任何学习器 OOB 覆盖，它的分数保持缺失，不能强行计入准确率。基础学习器数量增加后，覆盖通常会提高。

OOB 能利用训练过程自然产生的未见样本评价模型，常可减少单独验证集需求，但它不是测试集，也不意味着可以跳过最终独立评估。

## 函数职责

| 函数 | 输入 | 输出 | 下游用途 |
| --- | --- | --- | --- |
| `bootstrap_sample_indices` | 样本数、学习器数、种子 | 每个学习器的训练下标 | 重采样训练 |
| `out_of_bag_indices` | 一份采样下标、总样本数 | 未被采中的下标 | OOB 汇总 |
| `fit_bagging_stumps` | 训练数据和设置 | 学习器与采样记录 | 集成预测 |
| `oob_decision_function` | 模型、原训练特征 | OOB 分数与覆盖次数 | OOB 评价 |

参考实现还提供基础预测、普通决策分数和最终预测。学习时先按 starter 中四个函数建立采样与 OOB 主线。

## 最小样例

固定随机种子时：

```text
bootstrap_sample_indices(5, 2, random_state=0)
= [[4, 3, 2, 1, 1],
   [0, 0, 0, 0, 4]]
```

第一份采样的唯一入袋下标为 `{1,2,3,4}`，所以：

```text
OOB = [0]
```

第二份唯一入袋下标为 `{0,4}`，所以：

```text
OOB = [1,2,3]
```

样本0只能使用第二个学习器的 OOB 预测；样本1、2、3只能使用第一个学习器的 OOB 预测；样本4在这两轮都入袋，暂时没有 OOB 覆盖。

## 常见错误

- 使用无放回抽样，变成普通子采样；
- 每份 Bootstrap 只抽少于 `n` 个却仍称为标准 Bagging；
- OOB 下标通过“出现次数少”判断，而不是“完全没有出现”；
- 评价样本时混入见过该样本的基础学习器；
- 未覆盖样本按0分参与准确率；
- 所有学习器复用同一份 Bootstrap 下标；
- 忘记固定随机种子，demo 无法复现；
- 把 Bagging 误解为逐轮纠正上一轮错误。

4. 对一个训练样本列出哪些学习器有资格参与 OOB；
5. 比较学习器数量增加前后的 OOB 覆盖率；
6. 用一句话区分 Bagging 与 AdaBoost 的训练依赖关系。

能根据 `bootstrap_indices` 独立重建任意样本的 OOB 预测，才算真正掌握 OOB。
