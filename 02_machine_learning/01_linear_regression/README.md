# 线性回归

线性回归用数值特征预测一个连续数值，例如根据面积和房龄预测房价。

## 算法概览

线性模型试图学习特征的线性组合。对每个样本，它把各个特征分别乘上一个权重，再加上截距得到预测。虽然形式简单，线性回归是理解监督学习最合适的起点：模型、损失函数、参数估计、优化算法和泛化评价都能在这里形成完整闭环。

线性回归隐含的基本认识是：目标值可以由各特征的加性贡献近似描述。在其他特征保持不变时，一个权重表示对应特征增加一个单位会让预测改变多少，因此模型具有较强可解释性。但如果真实关系明显弯曲、存在复杂交互或异常值很多，普通线性关系可能不足，需要特征变换、正则化或更复杂模型。

从教材主线看，线性回归还是广义线性模型的基础。后面的逻辑回归会保留线性预测部分，再用联系函数把它转换成分类概率。

## 输入、输出与参数

给定特征矩阵 `X`，模型对全部样本计算：

```text
prediction = X @ w + b
```

- `X` 的每一行是一个样本，每一列是一个特征；
- `w` 为每个特征分配一个权重；
- `b` 是所有样本共享的截距；
- 预测负责用参数产生结果；
- 损失衡量预测与真实标签的差距；
- 训练通过解析解或迭代更新找到更合适的 `w, b`。

## 本阶段学习重点

当前先假定 `X、y、w` 都是题目给出的合法 NumPy 数组，不在学生代码中检查行数、列数、shape 或元素内容。本专题只练机器学习主线：

```text
预测 -> 计算损失/指标 -> 训练参数 -> 观察损失 -> 评价模型
```

数组形状、文件列数、缺失值和非法内容以后在文件读写与数据处理模块集中复验。

## 完整算法流程

```text
X, y
  |
初始化 w, b
  |
predict
  |
计算 error
  |
计算 MSE
  |
计算 gradient_w, gradient_b
  |
更新 w, b
  |
重复迭代
  |
返回参数和 loss history
```

最小二乘是另一条训练路径：在 `X` 左侧添加常数列，把截距并入参数，然后使用 `np.linalg.lstsq` 一次求解，不需要学习率、初始参数或迭代次数。

## 函数职责

| 函数 | 做什么 | 输入 | 输出 | 谁调用 | 为什么拆开 |
| --- | --- | --- | --- | --- | --- |
| `predict` | 计算 `X @ w + b` | `X, w, b` | prediction | 梯度、训练、最终预测 | 预测可独立复用和核对 |
| `mean_squared_error` | 计算平均平方误差 | 真实值、预测值 | 一个数值 | 训练、评估 | 把模型表现变成可比较的指标 |
| `mse_gradients` | 计算 MSE 对参数的梯度 | `X, y, w, b` | 两个梯度 | 梯度下降 | 告诉参数应往哪个方向更新 |
| `fit_gradient_descent` | 反复计算梯度并更新参数 | 训练数据与训练设置 | `w, b, loss_history` | 训练流程 | 展示模型如何逐步学到参数 |
| `fit_least_squares` | 用 `lstsq` 求最小平方误差解 | `X, y` | `w, b` | 训练、完整任务 | 与迭代训练形成对照 |

## 推荐实现顺序

```text
predict
-> mean_squared_error
-> mse_gradients
-> fit_gradient_descent
-> fit_least_squares
```

每完成一个函数就运行 `check.py`，不要等五个函数全部写完才定位第一个错误。

## 最小手算样例

使用：

```text
X = [[0], [1], [2]]
y = [1, 3, 5]
w = [0]
b = 0
learning_rate = 0.1
```

初始计算：

```text
prediction = [0, 0, 0]
error = prediction - y = [-1, -3, -5]
MSE = (1 + 9 + 25) / 3 = 35 / 3
X.T @ error = [0*(-1) + 1*(-3) + 2*(-5)] = [-13]
gradient_w = (2/3) * [-13] = [-26/3]
gradient_b = 2 * mean(error) = -6
```

一次更新后：

```text
new_w = 0 - 0.1 * (-26/3) = 13/15
new_b = 0 - 0.1 * (-6) = 3/5
```

最小二乘在这组精确直线上应得到 `w=[2]`、`b=1`。

## 常见错误

- 用 `*` 代替矩阵乘法 `@`；
- MSE 梯度漏掉 `2/n`，或对截距使用错误的求和方向；
- 学习率过大使损失发散；
- 更新了 `w` 却忘记更新 `b`；
- `loss_history` 没有保存初始损失，或保存时机错一位；
- 显式计算矩阵逆，而不是使用能处理秩亏的 `np.linalg.lstsq`。
