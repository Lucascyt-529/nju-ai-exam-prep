# 广播、循环变量与数组复制

## 出现场景

在实现按特征标准化时，需要把零标准差替换为1，同时保持调用者传入的原数组不变。

## 首次错误

1. 在 `for std in stds_safe` 中执行 `std = 1.0`，误以为会改回数组；
2. 随后把 `stds_safe = stds` 当成复制，导致布尔索引赋值同时修改原始 `stds`；
3. 初次中心化在函数内部重新计算当前矩阵均值，没有复用传入的训练统计量。

## 最小反例

```python
values = np.array([1.0, 0.0])

for value in values:
    if value == 0:
        value = 1.0

assert values[1] == 0.0
```

循环变量 `value` 只是本轮取出的标量，重新绑定它不会修改数组元素。

```python
safe_values = values
safe_values[safe_values == 0] = 1.0

assert values[1] == 1.0
```

赋值只建立别名；要保留原数组，需使用 `values.copy()`。

## 正确规则

- 用布尔索引修改数组中的目标元素；
- 需要独立副本时显式调用 `.copy()`；
- 验证集和测试集必须复用训练集拟合的均值与标准差，不能各自重新计算。

## 后续复验

在线性模型完整任务中再次检查训练统计量复用和输入不被原地修改；能完成变式后再视为稳定掌握。
