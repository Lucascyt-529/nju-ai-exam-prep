# 学习方法：从理解到迁移

## 当前基础

学习者目前只接触过少量 NumPy 和 pandas 基本语法，而且已经遗忘较多。教学不得默认学习者熟悉数组形状、索引、广播、`axis`、向量化或 DataFrame 操作。

历史目录显示这些内容过去曾经接触过，因此可以使用“诊断后加速”：先做最小预测与实现题，若能独立完成变式，则缩短重复讲解；若不能，则按零基础路径重建。不得因为旧文件存在而跳过诊断。

## 每个专题的六步循环

### 1. 建立直觉

先用2～5个可以手算的样本解释问题。首次出现的函数要说明输入、输出和形状，不直接堆叠长代码。

### 2. 运行前预测

在运行代码前，学习者先回答：

- 输出是什么；
- 数组形状是什么；
- `axis=0` 和 `axis=1` 分别在做什么；
- 修改一行输入后，结果会怎样变化。

预测错误不等于失败，它用来暴露真正不理解的地方。

### 3. 小步补全

先补一个局部步骤，例如解析一行、检查列数或计算一列均值。每次只引入少量新语法，并立即用最小样例验证。

默认以“一轮集中反馈”为节奏：Codex一次列出当前提交的全部问题和验收标准，学习者集中修改后统一测试。只有早期错误会让后续代码建立在错误概念上时，才临时拆成单步纠正。

### 4. 从空白独立实现

关闭参考实现，从 `starter.py` 或空文件完成完整程序。Codex只定位第一个关键问题，不立即重写答案。

### 5. 变式迁移

完成原题后至少做一个改变条件的任务，例如：

- 均值改为最大值或标准差；
- 标签从最后一列改到第一列；
- 输入增加表头、空行或缺失值；
- 输出从屏幕改为指定文件；
- 样本从二维改为单特征；
- 训练数据与测试数据采用不同分布。

如果只能复现原题而不能处理变式，不标记为掌握。

### 6. 解释与间隔复习

学习者用自己的话解释关键概念和错误原因。专题完成后安排隔日或跨专题复现，避免只依赖短期记忆。

## 掌握等级

| 等级 | 证据 | 记录方式 |
| --- | --- | --- |
| 见过 | 阅读过讲解或示例 | 不记为完成 |
| 能跟做 | 在提示下完成原题 | 记录为正在学习 |
| 能独立完成 | 不看答案通过原题测试 | 可更新为学生已独立完成原题 |
| 能迁移 | 独立完成至少一道变式并解释原因 | 才视为真正掌握 |

## 校验代码的考试优先级

- 必练：题目明确规定的格式、文件列数/类型、训练和标签样本数、关键二维/一维形状、缺失或非有限值要求；
- 按题目决定：空数据、类别范围、概率范围、矩阵可逆性等算法前置条件；
- 工程扩展：为每个内部小函数穷举错误类型、包装友好错误信息和重复的`isinstance`检查。

基础阶段用一个专题完整练会校验思想；后续限时算法训练默认相信题目保证的合法输入，只保留能防止静默错误或题目明确计分的检查。

## NumPy 与 pandas 的分工

### NumPy：考试与算法核心

NumPy 从零开始学习数组、形状、轴、索引、广播、矩阵计算、随机数和数值稳定。机器学习算法与考场核心程序默认使用 NumPy 手写。

### pandas：表格数据辅助能力

pandas 在 NumPy 基础建立后加入，覆盖：

- `Series` 和 `DataFrame`；
- `read_csv` 与 `to_csv`；
- 行列选择和布尔筛选；
- `dtypes`、缺失值和重复值；
- 排序、分组统计、合并与简单透视；
- pandas 与 NumPy 数组之间的转换。

pandas 不代替 NumPy 算法训练。若考试题禁止或环境未提供 pandas，核心解答必须仍能只依赖标准库和 NumPy 完成。

## 参考答案使用规则

1. 第一次作答不打开 `reference/`。
2. 测试失败时先阅读错误信息并构造更小样例。
3. Codex先提示错误位置和原因，不直接粘贴完整答案。
4. 对照答案后，隔一段时间从空文件重写一次。
5. `progress.md` 分开记录“看懂”“独立完成”和“完成变式”。
