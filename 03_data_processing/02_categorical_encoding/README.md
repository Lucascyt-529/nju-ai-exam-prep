# 数据处理2：类别词表、未知类别与one-hot

## 为什么不能直接把字符串改成任意整数

把`红/蓝/绿`编码为`0/1/2`只是在存储上使用整数，不表示类别之间存在数值距离。线性模型通常使用one-hot；树和贝叶斯方法可能直接处理离散取值，但仍需要稳定的训练词表。

最重要的隔离规则是：

```text
fit vocabulary：只查看训练集
transform：训练、验证、测试都复用同一词表
```

若测试集出现训练时未见的类别，不能偷偷重新拟合词表，否则列数和列含义都会改变。本专题提供两种明确策略：报错，或映射到每个特征块最后的`<UNKNOWN>`列。

## 形状推导

训练集有两个类别特征：

```text
颜色词表 = (红, 蓝)      -> 2个已知列 + 1个未知列
尺寸词表 = (大, 小, 中)  -> 3个已知列 + 1个未知列
```

则`X_categorical.shape==(n,2)`会变成`X_one_hot.shape==(n,7)`。one-hot的列数来自训练词表，不来自当前待转换数据。

## 学习顺序

1. 手工列出每个训练特征的排序词表；
2. 写出一个训练样本的序号编码；
3. 在纸上画出每个one-hot块的起止列；
4. 加入测试新类别，预测未知桶的位置；
5. 将数值特征`(n,d_num)`与one-hot`(n,d_cat)`按列拼接；
6. 将特征名、训练词表、未知桶开关保存为UTF-8 JSON；
7. 在新进程恢复元数据，改变测试数据后确认词表和输出列名不变。

运行引导演示：

```text
python 03_data_processing/02_categorical_encoding/guided_demo.py
```

## 易错点

- 用训练集和测试集的并集拟合词表，造成数据泄漏；
- 不保存列名，之后不知道第5列代表哪个类别；
- 把整数序号直接送入线性模型，错误引入大小关系；
- 未知类别全零却没有说明语义，和“所有类别都否”混在一起；
- 数值数组和one-hot拼接时样本顺序不一致。
- 只保存模型权重却不保存词表，使权重每一列的类别语义丢失。

## 迁移题

1. 关闭未知桶，确认新类别会明确报错；
2. 将三个类别特征的词表长度设为2、4、3，先算总列数再运行；
3. 输出所有one-hot列名，并逐列对照矩阵；
4. 设计单独的缺失类别`<MISSING>`，说明它与`<UNKNOWN>`的区别；
5. 将JSON中的`format_version`改成未知版本，确认加载器拒绝而不是猜测格式；
6. 把编码器JSON和NumPy模型包一起复制到另一目录，在新进程恢复后保持完全相同的列顺序和预测。
