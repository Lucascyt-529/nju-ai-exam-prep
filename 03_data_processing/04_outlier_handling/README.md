# 数据处理4：异常值诊断与稳健处理

## 先区分异常值与错误值

“离大多数样本很远”不等于“数据一定错了”。极端收入可能是真实少数样本，
也可能是单位录错；传感器尖峰可能是故障，也可能是需要识别的关键事件。
因此本专题把操作拆开：

```text
fit_iqr_bounds：只在训练集拟合诊断边界
detect_outliers：只标记，不改数据
summarize_outliers：统计哪些列、哪些行值得检查
clip_outliers：确认策略后才可选地裁剪
fit/transform_robust_scaler：用训练集中位数和IQR缩放
```

代码不提供“默认删除异常行”的函数。删除一行会同时改变所有特征和标签，
还可能系统性删除稀有类别；这必须由题意和领域规则决定。

## IQR边界

对每个特征按 `axis=0` 计算：

```text
IQR = Q3 - Q1
lower = Q1 - multiplier * IQR
upper = Q3 + multiplier * IQR
```

默认 `multiplier=1.5`。严格小于下界或严格大于上界才标为异常；恰好在边界上不标记。
这是统计诊断规则，不是关于数据真假的证明。

## 为什么只能在训练集fit

正确：

```text
X_train -> fit边界/中位数/IQR
X_train -> transform
X_valid -> 使用同一状态transform
X_test  -> 使用同一状态transform
```

错误：

```text
X_test -> 重新计算自己的分位数和边界
```

如果测试集整体比训练集大很多，重新拟合会让本应明显的分布漂移“看起来正常”，
也泄漏测试集分布。测试集缩放后中位数不必为0，这是正确现象。

## 稳健缩放不等于裁剪

稳健缩放使用：

```text
(X - training_median) / training_IQR
```

它降低均值和标准差对极端值的敏感性，但不会把异常值删除或限制到固定范围。
裁剪则会改变极端值本身。两者可以单独使用，也可按题意先裁剪再缩放，
但必须保存并复用训练阶段的全部参数。

训练列的 IQR 为0时，参考实现把缩放尺度替换为1，避免除零；这不会假装该列有变化。
IQR边界仍保持真实的零宽度，因此测试时出现不同值会被标记，提醒你检查分布变化。

## 学生任务顺序

1. 先完成 `fit_iqr_bounds`，用五个数手算Q1、Q3和上下界；
2. 完成广播比较，预测掩码形状；
3. 用 `axis=0` 和 `axis=1` 分别得到逐列计数与逐行标记；
4. 完成非原地裁剪；
5. 最后完成稳健缩放的fit/transform分离。

每一步先用小数组手算，再查看参考实现。不要背某一组边界数值。
