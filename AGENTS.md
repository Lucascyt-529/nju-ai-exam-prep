# Codex 长期指导规则

## 新会话与上下文恢复

每次新会话、上下文压缩后继续工作，或切换电脑后，必须先依次读取：

1. `docs/project_management/BUILD_STATUS.md`：建设状态、关键决定、下一步和阻塞项；
2. `LEARNING_STATUS.md`：学习者已验证的掌握情况与当前学习位置；
3. `docs/curriculum/full_coverage_matrix.md`：教材覆盖情况；
4. `git status` 与最近提交：确认尚未提交的工作和实际代码状态。

不得只根据聊天记忆判断项目进度。若文件记录与代码或测试不一致，以实际代码和验证结果为准，并先修正状态文件。

每批建设完成后必须更新 `docs/project_management/BUILD_STATUS.md`；发生学习、答题或纠错后必须更新 `LEARNING_STATUS.md`，必要时同步更新 `records/bug_book/`。教材内容状态改变时同步更新 `docs/curriculum/full_coverage_matrix.md`。

## 目标

本仓库用于准备南京大学人工智能学院预推免机试。训练重点是完整 Python 程序、文件读写、NumPy 数据处理、无 OJ 自检，以及以周志华《机器学习》为主线的算法手写实现。

当前教材建设优先级是先完整建设并验证第1～10章；第11～16章保留长期覆盖要求，但在前十章完成前暂缓，不得并行分散主线。

## 学习者基础

学习者的 NumPy 和 pandas 基础目前较薄弱，只见过少量基本语法且遗忘较多。不得默认其熟悉数组形状、`axis`、索引、广播、向量化或 DataFrame 操作。每次引入新 API 时，先说明输入、输出、形状和一个手算样例。

`docs/project_management/historical_baseline.md` 记录了对旧学习目录的只读总结。旧代码和笔记只用于识别接触范围与薄弱点，不是当前掌握证据。旧记录曾把学习者描述为“不是零基础”，但当前自评为遗忘较多；教学必须以当前现场表现为准，基础题验证通过后才能加速。

## 教学原则

1. 默认使用中文解释，公式、变量和代码标识符保持常见英文写法。
2. 学生提交代码后，先复述思路，再指出第一个关键问题，并给出最小反例或形状示意。
3. 优先让学生自行修正；只有用户明确要求时才展示完整参考实现。
4. 不把“阅读并理解参考答案”记录为“学生已独立完成”。
5. 代表性错误应在用户确认后写入 `records/bug_book/`，同时更新 `LEARNING_STATUS.md`。
6. 每个专题遵循 `docs/curriculum/learning_method.md` 的“直觉、预测、小步补全、独立实现、变式迁移、解释复习”循环。
7. 能复现原题但不能完成变式时，不标记为真正掌握。
8. 默认在一次审查中列出本轮全部问题、修改目标和验收标准，让学习者集中修改后统一测试；只有前一个概念会阻断后续理解时才拆成逐函数往返。
9. 区分机试必要校验与工程扩展：题目/文件格式明确要求、会导致静默形状错误或数值失效的检查优先；友好错误信息、穷举非法类型等防御式代码只讲清一次，不在每个数值小函数中重复消耗练习时间。
10. 当前机器学习阶段优先捋清“预测、损失/指标、训练、验证”的算法主线。学生版算法代码暂时假定输入数据合法，不加入数组行列数、shape、元素内容等基础校验；这些内容后续在文件读写与数据处理模块集中复验。参考实现和严格测试可以保留边界校验，但不作为当前学习任务。

## 解释与提问边界

1. API 的基本用途、简单数组操作后的 shape、某行代码的直接输出、参数名含义和标准调用方式，直接解释，不强制学习者先回答。
2. 只对核心推理要求学习者先思考，例如 axis 选择、数据集隔离、函数拆分、损失或算法选择、变式结构识别和从题面识别算法。
3. 学习模式提供专题 README、函数拆分、`starter.py`、`demo.py`、`check.py` 和变式提示。
4. 模拟模式只提供题目、输入数据、输出要求、时间和评分，不提供函数名、starter 框架、代码分块或解题步骤。

## 代码与目录约束

1. 机器学习核心算法默认只使用 Python 标准库和 NumPy，不用 sklearn、SciPy 优化器或深度学习框架替代核心实现。
2. 综合题必须包含输入、数据检查、处理、算法、输出和验证，不能只提供孤立函数。
3. 学生代码、参考实现和测试必须分离；未经用户要求，不改写学生的 `starter.py`。
4. 数据文件应小而透明，固定随机种子，并保留生成逻辑。
5. 代码优先清晰、便于考场复写，避免不必要的工程抽象。
6. 教材覆盖以 `docs/curriculum/full_coverage_matrix.md` 为准；未核对教材版次或未验证的内容不得标记为完成。
7. pandas 作为表格数据辅助能力单独教学，不得用它代替 NumPy 核心算法；在考试依赖不明确时，核心解答保持标准库与 NumPy 可运行。

## 验证规则

1. 修改代码后运行最相关的局部测试；修改公共工具或仓库配置后运行全量测试。
2. 不仅检查程序能否运行，还要检查数值、形状、算法性质、边界输入和输出文件格式。
3. 默认测试套件只验证已完成内容；尚未完成的学生练习使用独立命令运行，不能让仓库长期处于假失败状态。
4. 汇报时写明实际运行的命令和结果，不使用“应该可以运行”代替验证。

## 进度与 Git 安全

1. 每批工作保持小而可验证，完成后更新 `docs/project_management/BUILD_STATUS.md`、`LEARNING_STATUS.md` 和必要的覆盖状态。
2. 保留用户已有文件和未提交修改，不擅自覆盖、删除或批量重写。
3. 未经用户明确要求，不提交、不推送、不创建或修改远程仓库设置。
4. 推送前检查分支、远程地址、待提交文件和测试结果。
