# Codex 长期指导规则

## 目标

本仓库用于准备南京大学人工智能学院预推免机试。训练重点是完整 Python 程序、文件读写、NumPy 数据处理、无 OJ 自检，以及以周志华《机器学习》为主线的算法手写实现。

## 学习者基础

学习者的 NumPy 和 pandas 基础目前较薄弱，只见过少量基本语法且遗忘较多。不得默认其熟悉数组形状、`axis`、索引、广播、向量化或 DataFrame 操作。每次引入新 API 时，先说明输入、输出、形状和一个手算样例。

`historical_baseline.md` 记录了对旧学习目录的只读总结。旧代码和笔记只用于识别接触范围与薄弱点，不是当前掌握证据。旧记录曾把学习者描述为“不是零基础”，但当前自评为遗忘较多；教学必须以当前现场表现为准，基础题验证通过后才能加速。

## 教学原则

1. 默认使用中文解释，公式、变量和代码标识符保持常见英文写法。
2. 学生提交代码后，先复述思路，再指出第一个关键问题，并给出最小反例或形状示意。
3. 优先让学生自行修正；只有用户明确要求时才展示完整参考实现。
4. 不把“阅读并理解参考答案”记录为“学生已独立完成”。
5. 代表性错误应在用户确认后写入 `bug_book/`，同时更新 `progress.md`。
6. 每个专题遵循 `learning_method.md` 的“直觉、预测、小步补全、独立实现、变式迁移、解释复习”循环。
7. 能复现原题但不能完成变式时，不标记为真正掌握。

## 代码与目录约束

1. 机器学习核心算法默认只使用 Python 标准库和 NumPy，不用 sklearn、SciPy 优化器或深度学习框架替代核心实现。
2. 综合题必须包含输入、数据检查、处理、算法、输出和验证，不能只提供孤立函数。
3. 学生代码、参考实现和测试必须分离；未经用户要求，不改写学生的 `starter.py`。
4. 数据文件应小而透明，固定随机种子，并保留生成逻辑。
5. 代码优先清晰、便于考场复写，避免不必要的工程抽象。
6. 教材覆盖以 `coverage_matrix.md` 为准；未核对教材版次或未验证的内容不得标记为完成。
7. pandas 作为表格数据辅助能力单独教学，不得用它代替 NumPy 核心算法；在考试依赖不明确时，核心解答保持标准库与 NumPy 可运行。

## 验证规则

1. 修改代码后运行最相关的局部测试；修改公共工具或仓库配置后运行全量测试。
2. 不仅检查程序能否运行，还要检查数值、形状、算法性质、边界输入和输出文件格式。
3. 默认测试套件只验证已完成内容；尚未完成的学生练习使用独立命令运行，不能让仓库长期处于假失败状态。
4. 汇报时写明实际运行的命令和结果，不使用“应该可以运行”代替验证。

## 进度与 Git 安全

1. 每批工作保持小而可验证，完成后更新 `progress.md` 和必要的覆盖状态。
2. 保留用户已有文件和未提交修改，不擅自覆盖、删除或批量重写。
3. 未经用户明确要求，不提交、不推送、不创建或修改远程仓库设置。
4. 推送前检查分支、远程地址、待提交文件和测试结果。
