# 模拟机试1：基础闭环诊断

这是一套120分钟、150分的首次诊断卷。它的目的不是证明“前十章已经学完”，而是暴露从文件到结果的完整链路中最先卡住的位置。

## 正式计时前

1. 新建一个空白作答目录，把`student/`中的三个文件复制进去；
2. 只允许查看`PAPER.md`、`data/`和复制后的作答文件；
3. 不查看`reference/`、`expected/`、测试文件或Git历史；
4. 确认Python和NumPy可导入，但不要提前运行样例；
5. 准备纸或文本记录每题开始、暂停和结束时间。

## 计时策略

| 题目 | 建议时间 | 分值 | 到时策略 |
| --- | ---: | ---: | --- |
| 第1题 CSV统计 | 20分钟 | 30分 | 先保住读取、形状和输出表头 |
| 第2题 网格BFS | 30分钟 | 45分 | 先输出距离，再补路径 |
| 第3题 逻辑回归 | 70分钟 | 75分 | 先跑通训练预测，再补指标和保存恢复 |

每题最后至少留3分钟做自测。没有OJ时，必须主动构造一个极小反例，而不是只运行正式数据。

## 结束后

停止计时，保留当时文件，不立即覆盖。假设作答目录为`my_submission/`，运行：

```text
python grade_submission.py --submission my_submission
```

自动分数是初筛，不替代按`SCORING.md`审查源码；硬编码输出、绕过指定算法或隐藏泄漏必须在人工审查时扣回。评分后再查看参考实现或交给Codex审查，并在`REVIEW_TEMPLATE.md`记录：得分、用时、首个阻塞、形状错误、数据泄漏、边界遗漏和下次修正。
