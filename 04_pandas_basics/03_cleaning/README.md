# pandas 3：缺失值、重复值与类型转换

## 为什么分步骤清洗

真实 CSV 中的“缺失”可能表现为：

- 空字段；
- 只有空格的字符串；
- 本应是数字却写成 `bad`；
- `inf` 等不能用于模型的非有限数值。

本专题不使用一行 `dropna()` 把问题全部删除，而是明确执行：

```text
原始字符串读取
→ 去除首尾空格
→ 数值列转换（非法内容变为 NaN）
→ 按 sample_id 去重
→ 只在训练表拟合中位数
→ 使用已有中位数填补
→ 严格保存
```

## 关键 API

- `pd.read_csv(..., dtype=str, keep_default_na=False)`：保留原始文本状态；
- `.str.strip()`：清理字符串首尾空格；
- `pd.to_numeric(..., errors="coerce")`：无法转换的内容变为 `NaN`；
- `.duplicated()` / `.drop_duplicates()`：识别和删除重复记录；
- `.median()`：计算中位数；
- `.fillna()`：使用已经拟合的统计量填补。

`errors="coerce"` 不是“自动修好了数据”。它只是把问题统一显式标记为缺失，之后仍要决定拒绝、删除还是填补。
